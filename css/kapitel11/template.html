Kapitel 11 IDs und einfache Positionierung.

Trivia

Auf ganz ähnliche Weise wie Klassen funktionieren IDs: Auch hier Vergeben Sie einen selbstdefinierten Namen, den Sie dem gewünschten HTML-Element zuweisen , um dieses von anderen, gleichartigen HTML-Elementen zu unterscheiden. In den CSS-Vorgaben ändert sich lediglich der Punkt in eine Raute, und das HTML-Attribut class wrid durch id ersetzt:

Beispiel

CSS:
p#graue_box{background-color: gray;}

HTML:
<p id="graue_box">Absatz mit grauem Hintergrund.</p>

Auch eine ID können Sie jedem beliebigen HTML-Element zuweisen.

Was ist nun aber der Unterschied zwischen Klassen und IDs? Er liegt in der Logik: Wie wir gesehen haben, können Sie eine bestimmte Klasse so oft und so vielen Elementen auf Ihrer Seite zuweisen, wie Sie möchten. Eine bestimmte ID hingegen darf nur ein einziges Mal in einem HTML-Dokument vorkommen. Dadurch ist das betreffende Element gegenüber anderen Elementen eindeutig identifizierbar. Dabei spielt es keine Rolle, welchem HTML-Element Sie die ID zuweisen. Wichtig ist lediglich, dass nur ein einziges Element diese bestimmte ID trägt.

IDs verwenden Sie also immer dann, wenn Sie wissen, dass ein bestimmtes Gestaltungselement nur einmal auf jeder Ihrer Seiten vorkommt. Klassen hingegen nutzen Sie für Elemente, die in Ihrem Layout mehrfach vorkommen.

Aus diesem Grund dienen IDs meist dazu, das Layout einer Seite vorzugeben und sie in verschiedene Bereiche aufzuteilen(Kopfbreich, Inhaltsbereich), während der Inhalt der Seite eher mit Klassen formatiert wird.

IDs werden überdies häufig benötigt, wenn die betreffenden Elemente mit Hilfe von Javascript manipuliert werden sollen.

HTML-Tags für das Seitenlayout.

Dieses "Layoutraster" wurde in der Vergangenheit in HTML häufig mit Hilfe von div-Tags erzeugt(ist aber keineswegs auf diese beschränkt). Der div-tag ist ein Blockelement und erzeugt daher - wie etwa der p-Tag - eine neue Zeile. Er hat gegenüber dem p-Tag jedoch einen entscheidenden Vorteil: er kann weiter Blockelemente enthalten Dadurch können sie mit Div-Elementen gesonderte, in sich abgeschlossene Breiche oder Abschnitte definieren, die selbst weitere Elemente wie. Überschriften, Absätze, Listen, Tabellen, aber auch Bilder und sogar weitere div-Elemente enthalten können. Mit CSS formatieren und positioniert können Sie damit jedes Layout realisieren; Navigationsleisten, Textspalten, Infoboxen.

Neue Tags in HTML-5

In früheren HTML-Versionen war der div nahezu der einzige, mit dem ein "Layoutraster" möglich war. Ein Nachteil dabei ist, dass den einzelnen Seitenbereichen keine semantische Bedeutung zukommt. Ob Kopfbereich, Hauptnavigation oder Inhalt - alles sind div-Container. Seit HTML 5 gibt es zu diesem Zweck zusätzliche Tags, die die einzelenen Seitenbreiche von vornherein semantisch als solche kennzeichnen.

Semantische HTML-5 Tags

<header>

definiert eine Einleutung oder den Kopfbereich eines Dokuments, aber auch eines einzelen Artikels. Es kann z.B Überschriften, aber auch Navigationselemente enthalten. Nicht zu verwechseln mit <head>, dem(unsichtbaren Dateikopf am Beginn jedes HTML-Dokuments)

<nav>

kennzeichnet die darin enthaltenen Links als Navigationselemente

<main>

kennzeichnet den Haupt-Inhaltsbereich.

<footer>

definiert eine Fußzeile. Sie kann z.B. den Autor des Dokuments, Copyright-Informationen, Links zu Nutzungsbedingungen, Kontaktdaten etc. enthalten. Fußzeilen werden in der Regel am unteren Rand des Dokuments platziert(Dies ist jedoch nicht zwingend erforderlich). Ein Dokument kann auch mehr als eine Fußzeile enthalten.

<section>

kennzeichnet einen größeren, in sich geschlossen Bereich oder Abschitt, z.B den Bereich unterhalb eines Blog-Beitrages, in dem die User-Kommentare stehen. Wird bis jetzt eher selten verwendet.

<article>

kennzeichnet einen unabhängigen,in sich geschlossen Abschnitt(i.d.R innerhalb des Inhaltsbereichs) z.B einen einzelnen Foren- oder Blog-Beitrag, einen User-Kommentar oder die Beschreibung eines bestimmten Produkts in einem Online-Shop.

<aside>

definert inhalt, der zum jewiligen Thema passt, aber nicht direkt dazugehört, eine Randbemerkung oder Marginalie. Kann auch zur Erzeugung von Seitenleisten verwendet werden.

NOTE: Beachten Sie, dass diese sogenannte "strukturellen Tags" ohne CSS-Vorgaben keinerlei optische Veränderung bewirken, Sie dienen lediglich der logischen Aufteilung Ihres Dokuments

NOTE: Der Internet Explorer bis zur Version 8 hat die Eigenschaft, HTML-Tags, die er nicht kennt, überhaupt nicht darzustellen, selbst wenn diesen mit CSS ein eindeutiges Aussehen und eine eindeutige Position zugewiesen wurde. Deshalb funktionieren die genannten HTML-Layouts in IE erst ab Version 9, da erst diese Version HTML-5 fähig ist. Um die neuen Tags auch in älteren IE-Versionen darstellen zu können, benötigen Sie spezielle Hilfsmittel, sogenannte Polyfills.(Siehe Kapitel 14.)
