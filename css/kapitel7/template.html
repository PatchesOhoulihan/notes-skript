Kapitel 7 Hintergründe, Konturen und Abstände.

Hintergrundfarben

Sie können mit CSS für jedes beliebige HTML-Element eine Hintergrundfarbe definieren. Das funktioniert sowohl für Blockelemente als auch für Inline-Elemente.

p{background-color:#d48e80}
a:hover{background-color: #3b8ed8}

Nun sehen sie deutlich eine der Haupteigenschaften von Blockelementen: Sie nehmen immer die gesamte zur Verfügung stehende Breite in Anspruch, auch wenn der Text selbst wesentlich kürzer ist.

Innenabstände

Den Innenabstand eines Elements können Sie mit dem Attribut padding und der Angabe von nummerischen Werten beeinflussen.

padding: 5px;

Auch diese Werte können Site stattdessen relativ zur Schriftgröße angeben:

padding: 0.3em; oder,
padding 0.3rem;

Sie können auch für jede Seite individuelle Innenabstände definieren.

padding-top: 0;
padding-bottom:0.3em;
padding-left: 0.7em;
padding-right: 1em;

Geben sie alle vier Werte - durch ein leerzeichen getrennt - im Attribute padding an. Die Reihenfolge hierbei ist im Uhrzeigersinn: oben rechts, unten, links

padding: oben rechts unten links;
padding 0 1em 0.3em 0.7em

Werte für sind natürlich meist nur dann Sinnvoll, wenn das Element eine sichtbare Kontur oder eine Hintergrundfarbe hat.

Einbinden von Hintergrundbildern

Ebenso universell wie Hintergrundfarben lassen sich für jedes Element Grafiken als Hintrgrundbilder einbinden.

background-image: url(pfad/zur/grafik.jpg)

Als Dateiformate können Sie entweder Pixelbilder in den bekannten Formaten JPEG, PNG, oder GIF oder Vektorengrafiken im SVG-Format verwenden

Kachelmuster steuern

Mit CSS lässt sich die Dartstellung von Hintergrundbildern sehr flexibel steuern. So können Sie z.B die automatische Widerholung abschalten:

p{background-image: url(grafik.jpg);}
background-repeat: no-repeat;

Mögliche Werte für background-repeat.
background-repeat: top(oben bündig)
bottom(unten bündig)
center(horizontal oder vertikal zentriert)
left(linksbündig)
right(rechtsbündig)

Hintergrundbild positionieren

Mit background-position lässt sich die Position des Hintergrundbilds numerisch festlegen. Geben sie dazu zwei Werte durch Leerzeichen getrennt ein. Der erste Wert ist der horizontale Abstand von links, der zweite is der vertikale Abstand von oben:

p{background-image: url(grafik.jpg);
background-repeat: no-repeat;
background-position: 1.5em 3em;}

Mögliche NICHT nummerische Werte sind
background-position: top(oben Bündig)
bottom(unten bündig)
center(horizontal oder vertikal zentriet)
left(linksbündig)
right(rechtsbündig)

Wasserzeicheneffekt

Mit dem Arttibut background-attachment können Sie festlegen, dass das Hintergrundbild beim Scrollen der Seite stehenbleibt. der Text scrollt also über das Bild hinweg.

p{background-image: url(grafik.jpg)
background-repeat: no-repeat;
background-position: 1.5em 3em;
background-attachment: fixed;}

background-attachment: scroll(mitscrollen, Standardeinstellung)
fixed(Hintergrundbild bleibt stehen.)

Eigenschaften zusammenfassen

Alle genannten Eigenschaften für Hintergründe können auch mit dem "Sammelattribut" background nach dem folgenden Muster angeben. Die einzelnen Werte werden dabei durch Leerzeichen getrennt. Die Reihenfolge spielt - bis au die beiden Werte für background-position - keine Rolle.

Syntax: background: color url repeat position-x position-y attachment

Dabei müssen Sie nicht zwingend alle Werte angeben

Größenangaben für Hintergrundbilder

Seit CSS 3 können Sie auch endlich die Größe von Hintergrundbildern bestimmen. Das Attribut dafür ist background-size (zurzeit noch mit Vendor Prefix) und erwartet zwei Werte.(Breite und Höhe) durch leerzeichen getrennt:

-webkit-background-size:breite höhe;
-moz-background-size: breite höhe;
background-size: breite höhe

-webkit-background-size: 2em 1em;
-moz-background-size: 2em 1em;
background-size: 2em 1em;

In den meisten Fällen sind hier wohl Prozentangaben am sinnvollsten (bezogen auf die Größe des Elements, das das Hintergrundbild enthält). Um das Seitenverhältnis beizubehalten und das Bild nicht zu verzerren, sollten Sie zudem einen der beiden Werte auf auto setzen:

-webkit-background-size:auto 70%;
-moz-background-size: auto 70%;
background-size: auto 70%

Automatische Größenanpassung von Hintergrundbildern

Im Zusammenhang mit flexiblen Layouts, deren Größe sich an das jeweilige Gerät anpasst, sind vor allem die beiden Schlüsselwörter contain und cover nützlich,

  • background-size: contain

    passt das Hintergrundbild automatisch so in das Element ein, dass es immer vollständig sichtbar ist U.U entstehen dadurch auch Leerräume, an dennen die Hintergrundfarbe durchscheint

  • background-size:cover

    passt das Hintergrundbild so ein, dass es das Element auf jeden Fall vollständig ausfüllt. Ggf werden dadurch teile des Hintergrundbildes abgeschnitten. Welche Teile das sind , können Sie mit background-postion steuern.

    Diese automatischen Einpassungen werden auch bei einer Größenänderungen des Browserfenster

    Diese Eigenschaft background-size lässt sich zurzeit noch nicht verlässlich in das "Sammelattribute" background integrieren und muss daher separat notiert werden.

Here Be Dragons: Farbverläufe in CSS

Eine der beeindruckendsten, gleichzeitig aber auch der komplexesten Funktionen in CSS 3 sind wohl die Farbverläufe, die in früheren CSS-Versionen ebenfalls mit Pixelbildern realisiert werden mussten. Die Vorteile liegen auf der Hand: Im Gegensatz zu Pixelbildern realisiert werden mussten. Die Vorteile liegen auf der Hand: Im Gegensatz zu Pixelbildern haben in CSS definierte Farbverläufe keine feste Größe, d.h sie passen ihre Größe - ohne Interpolation - automatisch an die Größe des Elements an. Außerdem muss bei einer Farbänderung nicht jedesmal die Pixelgrafik neu erzeugt und gespeichert werden, sondern es reicht eine kleine änderung in CSS

Einen Farbverlauf weisen Sie in CSS ebenfalls mit der Eigenschaft background-image zu. Als Wert folgt dann jedoch anstelle der bisher bekannten URL Definition des Verlaufs. Wie in den meisten Grafikprogrammen haben Sie auch hier die Wahl zwischen einem linearen und einem radialen Verlauf:

background-image: linear-gradient();
background-image: radial-gradient();

Ärgerlicherweise müssen auch Farbverläufe zurzeit noch mit Vendor Prefixes versehen werden. Diesmal jedoch nicht für die Eigenschaft, sondern für den Wert:

background-image: -webkit-linear-gradient()
background-image: -moz-linear-gradient()
background-image: linear-gradient()

background-image: -webkit-radial-gradient()
background-image: -moz-radial-gradient()
background-image: radial-gradient()

Lineare Farbverläufe

Die grundlegende Syntax für lineare Farbverläufe sieht folgendermaßen aus(ist aber um zusätzliche Optionen erweiterbar):

background-image: linear-gradient(Winkel, Anfangsfarbe, Endfarbe)

NOTE: Die Angaben zum Verlaufswinkel werden in den browserspezifischen Varianten anders interpretiert als in der standardkonformen Variante!

Beispiele

background-image: -webkit-linear-gradient(0deg, red, blue);
background-image: -moz-linear-gradient(0deg, red, blue)
background-image: linear-gradient(90deg, red, blue)

erzeugt einen horizontalen Verlauf von links(rot) nach rechts(blau)

background-image: -webkit-linear-gradient(180deg, red, blue);
background-image: -moz-linear-gradient(180deg, red, blue)
background-image: linear-gradient(-90deg, red, blue)

erzeugt einen horizontalen Verlauf von rechts(rot) nach links(blau)

background-image: -webkit-linear-gradient(90deg, red, blue);
background-image: -moz-linear-gradient(90deg, red, blue)
background-image: linear-gradient(0deg, red, blue)

erzeugt einen vertikalen Verlauf von unten(rot) nach oben(blau)

background-image: -webkit-linear-gradient(45deg, red, blue);
background-image: -moz-linear-gradient(45deg, red, blue)
background-image: linear-gradient(45deg, red, blue)

erzeugt einen vertikalen Verlauf von unten links(rot) nach oben rechts(blau)

background-image: -webkit-linear-gradient(-90deg, red, blue);
background-image: -moz-linear-gradient(-90deg, red, blue)
background-image: linear-gradient(180deg, red, blue)

erzeugt einen vertikalen Verlauf von oben(rot) nach unten (Blau). Dies ist in allen Varianten Die Standardeinstellung d.h in diesem Fall kann die Angabe für den Winkel weggelassen werden:

background-image: -webkit-linear-gradient(red, blue);
background-image: -moz-linear-gradient(red, blue)
background-image: linear-gradient(red, blue)

NOTE Anstelle des Winkels können auch Schlüsselwörter wie z.B left, top, Bottom ,right verwendet werden. Aber auch diese verhalten sich in den unterschiedlichen Varianten inkonsistent

Farbangaben

Für die Farbwerte können Sie jede beliebige Notation verwenden. Am häufigsten sind wohl Hex-Werte, aber es sind sogar Alphatransparenzen mit RGBA-Werten möglich:

background-image: linear-gradient(0deg, #fff, rgba(255,255,255,0.5))

erzeugt einen vertikalen Verlauf von unten (weiß deckend) nach oben(weiß mit 50%-iger Deckkraft)

Sie können natürlich auch mehr als zwei Farben angeben:

background-image:linear-gradient(90deg, red, orange, yellow, green, blue, indigo, violet)

erzeugt einen horizontalen Verlauf von links nach rechts in Regenbogenfarben.

Color Stops

Und schließlich können Sie mit sogenannten Color Stops die Postion der Farbübergänge verändern. Color Stops werden - nach einem Leerzeichen - als Prozentwert hinter die betreffende Farbe geschrieben

background-image: linear-gradient(90deg, red, blue 15%)

erzeugt eine horizontalen Verlauf von links nach rechts, wobei der übergang von Rot nach Blau bereits nach 15% des Verlaufs erfolgt, und nicht - wie standardmäßig - erst nach 50%

Radiale Farbverläufe

Die allgemeine, standardkonforme Syntax für radiale Verläufe lautet:

background-image: radial-gradient(Form at Position-x Position-y,Anfangsfarbe, Endfarbe)

wobei hier die Optionen für Form und Position optional sind. Werden sie weggelassen, entsteht ein Verlauf, dessen Zentrum im Mittelpunkt des Elements steht, und der seine elliptischen form an die des Elements anpasst z.B

background-image: radial-gradient(red, blue);

Der Standardwert für die Form ist eine ellipse Mit dem Wert circle lässt sich eine Kreisform erzwingen:

background-image:radial-gradient(circle, red, blue)

Zusätzlich können Sie mit der Option at Position-x Position-y den Mittelpunkt des Verlaufs verschieben

background-image: radial-gradient(circle at 1em 1em, red, blue)

Analog zum linearn Farbverlauf lassen sich natürlich auch hier die Anzahl der Farben sowie de Color Stops ändern.

NOTE: Auch beim radialen Verlauf hat sich zwischenzeitlich die Syntax geändert! die browserspezifischen Angaben mit Präfix müssen Sie noch nach der alten Syntax notieren:

background-image:-webkit-radial-gradient(Position-x Position-y, Form, Anfangsfarbe, Endfarbe)
-moz-radial-gradient(Position-x Position-y, Form, Anfangsfarbe, Endfarbe)

Somit müsste die vollständige Angabe eines radialen Verlaufs z.B so aussehen:

background-image:-webkit-radial-gradient(1 em 1 em, circle, red, blue)
-moz-radial-gradient(1 em 1 em, circle, red, blue)
background-image: radial-gradient(circle at 1 em 1 em, red, blue)

Zudem existiert noch eine weitere, völlig andere Syntax zur Erzeugung linearer und radialer Farbverläufe speziell für Webkit-basierte Browser, der Sie im Web vielleicht noch hier und da begegnen:

background-image: -webkit-gradient(linear, left top, left bottom, color-stop(0,red), color-stop(1,blue))

Diese experimentelle Syntax stammt noch aus den Anfangszeiten von CSS 3 und ist veraltet.

Aufgrund der unterschiedlichen Syntax, sowie die wahrscheinlichkeit das sich diese noch ändert. Aus diesem Grund gibt es einige Website auf dennen man sich den Code generieren lassen kann.

www.colorzilla.com/gradient-editor
www.cssmatic.com/gradient-generator
http://cssfactory.com/linear-gradient

Festlegen von Konturen

Die können jedem beliebigen Objekt auf ihrer Seite eine Kontur verleihen Dazu stehen Ihnen drei Optionen zur Verfügung:

  1. borderwidth Konturstärke

  2. border-style Randstil

  3. border-color Konturfarbe

Konturstärke

Mit h1 {border-width: 1px;} definieren Sie die Konturstärke i.d.R in Pixeln. Sie können jedoch auch für jede der vier Seiten eines Objekts separate Konturstärken definieren:

h1{border-top-width: 1px
border-right-width: 2px;
border-bottom-width: 3px;
border-left-width: 4px;}

oder

Die Reihenfolge hierbei ist wieder oben beginnend im Uhrzeigersinn.

NOTE: Die Werte, die Sie für border-width angeben, werden in der Gesamtbreite des Elements hinzugefügt.

Randstil

Mit border-style definieren Sie den Rahmentyp. Mögliche Werte sind:

borderstyle:

solid (durchgezogen)

dotted(gepunktet)

dashed (gestrichtelt)

double (doppelt)

groove (3D-Effekt)

ridge (3D-Effekt)

inset (3D-Effekt)

outset (3D-Effekt)

none (keine Kontur, sinnvoll zum Abschalten vorhandener Konturen)

Alle Kontureigenschaften auf einmal definieren

vereint die properties: border-width border-style und border-color

h1{border: 1px solid red}

Außenabstände

Viele Blockelemente -wie zum Beispiel etwa p oder h1 - erzeugen in der Browserdarstellung bekanntlich einen Abstand noch oben und unten. Diesen Abstand können Sie mit dem Attribut margin beeinflussen

p{margin: 1em;}

Die Angabe von margin wirkt sich jedoch wieder auf alle vier Seiten es Elements aus. Sie können auch für jede Seite individuelle Abstände definieren.

p{
margin-top: 0;
margin-bottom: 1em;
margin-left: 0.5em;
margin-right: 0.5em;

Oder Sie geben alle vier Werte, durch Leerzeichen getrennt an(beginnend im Uhrzeigersinn(start mit oben))

p{margin: 0 0.5em 1em 0.5em;}

In der Praxis wären für Überschriften und Absätze z.B folgende Angaben sinnvoll:

p,h1,h2,h3,h4,h5,h6{
margin: 0 0 1rem 0;
}

Damit deaktivieren Sie die automatischen Abstände noch oben und fügen einen genau definierten Abstand nach unten ein.
