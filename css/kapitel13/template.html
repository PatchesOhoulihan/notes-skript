Kapitel 13 Ändern der Anzeigeart und kontextbedingte Auswahl.

Ändern der Anzeigeart von Elementen

Wie wir wissen, gibt es in HTML grundsätzlich zwei Arten von Elementen:

  1. Blockelemente, die einen Absatz erzeugen(div, p, h1-h6, header, main, footer nav....)

  2. Inline-Elemente, die innerhalb des Fließtextes benutzt werden können(z.B. strong, em, b, i, a, sup....)

Mit CSS können Sie diese Darstellungsart eines Objekts ändern. Dazu benutzen Sie das Attribut display. Folgende Angaben sind möglich:

Attribute für display

display:

block (wandelt ein Inline-Element in ein Blockelement um - das Element erzeugt eine neue Zeile.)

inline (wandelt ein Blockelement in ein Inline-element um - das Element wird im laufenden Textfluss angezeigt)

inline-block (erzeugt einen Block, für den Breite, Höhe, und Außenabstand angegeben werden können, ) belässt das Element jedoch im laufenden Text, ähnlich einem Bild(img)

list-item(wie block, jedoch mit einem Listenpunkt davor)

none(Das Element wird nicht angezeigt und nimmt auch keinen Raum mehr in Anspruch.)

Zudem haben Sie die Möglichkeit, Elemente so darzustellen, als wären sie Tabellen. Dadurch können Sie die Vorteile von Tabellen zu Layoutzwecken nutzen, ohne in HTML tatsächlich Tabellen zu verwenden:

display

table
table-row
table-cell

Positionsbedingte Auswahlen

Descendant Selector(Nachfahren-Selector)

Es ist nicht unbedingt notwendig, für jedes Element auf ihrer Seite, das gesondert formatiert werden soll, gleich eine eigene Klasse oder ID zu definieren. Sehr oft ist es einfacher(und eleganter), Elemente abhängig von ihrer Position zu formatieren.

Sie können Elementen unterschiedliche Eigenschaften zuweisen, abhängig davon, in welchem übergeordneten Element(Ancestor, "Vorfahr") sie enthalten sind. Notieren Sei dazu einfach in einem neuen Selektor zuerst das Vorfahren-Element und nach einem Leerzeichen das Nachfahren-Element, das sie formatieren möchten(Descendant)

nav a{
display:block
}

Diese bedingten Formatierungen können mit allen Arten von CSS-Selektoren und in beliebiger Tiefe verwendet werden.

Beispiele

div p

ändert alle Absätze <p>, die sich innerhalb eines beliebigen div-Tags befinden, nicht aber die Absätze außerhalb von div-Tags

.box h1

ändert alle Überschriften <h1>, die sich innerhalb von Elementen mit dem Klassenamen .box befinden

div#nav ul

ändert alle Listen <ul>, die sich innerhalb des div-Containers mit der ID #navi befinden.

div#menu ul.rot li

ändert alle Listeneinträge<li> in ul-Listen mit dem Klassennamen .rot, sofern sich diese in einer div-Box mit der ID #menu befinden.

NOTE: Diese art der Selektion ist in vielen Fällen effizienter als die Vergabe unzähliger Klassen und IDs Bedenken Sie bei der Verwendung solcher Selektoren immer folgende Regel: Es werden immer nur die Elemente formatiert, die ganz rechts in der Zeile stehen. Im Beispiel:

div#menu ul.rot li

wäre das also ausschließlich das li-Element Weder die ul-listen mit der Klasse rot noch das div#menu sind von der Formatierung betroffen.

Child Selector(Kind-selector)

Eine Variante der Nachfahren-Selektoren sind die Kind-Selektoren, mit denen Sie sehr komfortable Elemente formatieren können, die direkte "Kinder" von übergeordnenten Parent-Elementen sind.

Die allgemeine Syntax lautet:

parent > child

wobei der "Pfeil" mit dem Zeichen für "größer als" gebildet wird.

Beispiele

div > p

ändert alle Absätze <p>, die sich direkt innerhalb eines beliebigen div-Tags befinden, nicht aber die Absätze, die sich innerhalb weiterer Elemente befinden:

<div>
   <p></p>
</div>

wird geändert!

<div>
  <article>
    <p></p>
  <article>
</div>

wird NICHT geändert

.box > h1

ändert alle Überschriften <h1>, die sich direkt innerhalb von Elementen mit dem Klassennamen .box befinden:

<div class="box">
   <h1></h1>
</div>

Wird geändert.

<div class="box">
  <article>
    <h1></h1>
  <article>
</div>

wird NICHT geändert

ul#navi > li

ändert alle Listenelemente <li>, die sich direkt innarhalb der <ul>-Liste mit der ID#navi befinden:

<ul id="navi">
   <li></li>
</ul>

Wird geändert.

<ul id="navi">
  <li>
      <ul>
        <li></li>
      </ul>
  </li>
</ul>

wird NICHT geändert

ul#menu > li > a

ändert alle Links auf der obersten Ebene einer <ul>-Liste mit der ID #menu, nicht aber die Links in verschachtelten Listen:

<ul id="menu">
  <li>
    <a></a>
  </li>
</ul>

<ul id="menu">
  <li>
      <a></a>
      <ul>
        <li>
          <a></a>
        </li>
      </ul>
  </li>
</ul>

ändert nur die äußeren Links, nicht aber die in der verschachtelten Liste.

Adjacent Sibling Selector(Selector für "benachbarte Geschwister")

Mit diesem Selektor können Sie Elemente auswählen, die im HTML-Quelltext unmittelbar nach einem bestimmten Element(nicht innerhalb) stehen. Der Kombinator ist hier das Pluszeichen. Die allgemeine Synstax ist:

element1 + element2

Beispiel: h1 + p{font-weight: bold;} ⇒ formatiert alle Absätze, die im HTML-Quelltext direkt nach einer h1-Überschrift stehen. Im folgenden Beispiel ist dies ausschließlich der erste Absatz:

<h1>Überschrift</h1>

<p>Nur dieser Absatz wird formatiert.</p>

<p>Zweiter Absatz</p>

<p>Dritter Absatz</p>

Leerzeichen im HTML-Quelltext spielen dabei natürlich keine Rolle. Sie werden ignoriert.

General Sibling Selector(Selector für alle nachfolgenden Geschwister.)

Dieser Selector ist mit Vorsicht zu genießen, denn er formatiert alle genannten Elemente die im HTML-Quelltext auf ein bestimmtes Element folgen. Der Kombinator ist die ~ (Tilde)

element1 ~ element2

Beispiel h1~p{font-weight: bold;}

formatiert alle Absätze, die nach der h1- Überschrift auf derselben Ebene stehen:

<h1><>h1<Überschrift/h1></h1>
<p>Dieser Absatz wird formatiert.</p>
<p>Dieser Absatz wird formatiert</p>
<p>Dieser Absatz wird formatiert</o>
<div>
  <p>Dieser Absatz wird nicht eingefärbt </p>
</div>

Auch hier werden die Leerzeichen natürlich ignoriert.

Attribute-Selektoren

Anders funktionieren die Attribut-Selektoren: Damit können Sie Elemente abhängig davon auswählen, ob sie im HTML-Dokument bestimmte Attribute besitzen. Attribut-Selektoren stehen in eckigen Klammern.

Einfacher Attribute-Selektor

[attribute]

Beispiel

[src]

Dieser Selektor wählt alle Elemente aus, die das HTML-Attribut src besitzen. Wie zum Beispiel:

<img src="deinemama.jpg"/>
<video src="film.mp4"></video>

Tag mit Attribut-Selector

tag[attribut]

Beispiel

input[type]

wählt alle input-Elemente aus, die das Attribut type haben

<input type="text"/>
<input type="checkbox"/>
<input type="submit"/>

aber NICHT

<button type="submit"/>

Wertabhängige Attribut-Selektoren

Operatoren

Mit verschiedenen Operatoren anstelle des Gleichheitszeichens können Sie die Auswahl ausweiten:

Der ^= Operator wählt alle Elemente aus, die ein Attribute haben, dessen Wert mit bestimmten Zeichen beginnt:

a[href^=http:]

wählt alle Links aus, deren Wert im href-Attribut mit http: beginnt z.B:

<a href="http://www.youtube.de">

Aber NICHT

<a href="../products/kaffeemachine.html">

Der $= Operator wählt alle Elemente aus, die ein Attribut haben, dessen Wert mit einer bestimmten Zeichenfolge endet

img[src$=".png"]

wählt alle Bilder aus, deren Wert im src-Attribut mit .png endet.

Der *= Operator wählt alle Elemente aus, die ein Attribut haben, dessen Wert ein bestimmtes Zeichen enthält

div[id*="navi"]

wählt alle div-tags aus deren, ID die Zeichenkette navi enthält.

<div id="navi">
<div id="hauptnavigation">
<div id="subnavi">
<div id="navigationsmenu">
<div id="navi_unten">

Der Stern Selektor

Die Holzhammer-Methode schließlich ist der Stern-Selektor *. Dieser Selektor wählt alle Elemente aus. Er wird in der Praxis häufig dazu verwendet, um z.B. das Standard-margin des Browsers von sämtlichen Elementent im Dokument zu entfernen:

*{
  margin: 0;
}

In diesem Fall müssen natürlich allen Elemtnen(vor allen Absätzen und Überschriften) die gewünschten margins manuell wieder hinzugefügt werden, Dennoch ist dieses Vorgehen meist effektiver als der umgekehrte Weg, das margin von jedem einzelnen element zu entfernen oder anzupassen.
