Kapitel 6 - Pseudoklassen und Pseudoelemente.

1. Pseudoklassen

Mit Hilfe sogenannter Pseudoklassen können Sie Vorgaben anhand von Informationen definieren, die entweder außerhalb der Dokumentstruktur liegen oder mit anderen Selektoren nicht ausgedrückt werden können, z.B ein Link, den der Anwender mit der Maus überfährt

Pseudoelemente stehen nicht als eigene Tags im Quelltext eines HTML-Dokuments, werden von CSS aber so behandelt, als stünden sie darin z.B "der erste Buchstabe eines Absatzes"

Verschiedene Zustände für Hyperlinks definieren

Sie können für Hyperlinks verschiedene Darstellungen definieren, je nachdem, in welchem Zustand sie sich gerade befinden. Ein Hyperlink kann folgende Zustände annehmen:

  • normal(noch nicht besucht)

  • bereits besucht

  • mit der Tabulatortaste fokussiert

  • Maus darüber

  • Maus gedrückt

Dies Zustände definieren Sie mit Hilfe speziell dafür vorgesehener Pseudoklassen Diese werden mit einem Doppelpunkt einfach an den Selektor angehängt(ohne Leerzeichen).

  1. a:link(noch nicht besuchte Seiten)

  2. a:visitied(bereits besuchte Seiten)

  3. a:focus(mit der Tabulatortaste fokusiert.)

  4. a:hover(Maus darüber)

  5. a:active(Maus gedrückt)

Beachten Sie, dass dies KEINE frei wählbaren Namen sind, sondern fest vorgegebene Schlüsselwörter. Bei der Definition dieser Pseudoklassen sollten Sie o.g Reihenfolge einhalten, da später notierte Angaben doe zuvor notierten überschreiben. So könnte sich z.B a:hover auf einem mit der Tastatur ausgewählten Link nicht auswirken, wenn a:focus später notiert wird code

Vor allen der Zustand a:visited führt häufig zu Missverständnissen: Dieser Zustand kennzeichnet nicht die Seite, auf der sich der Besucher gerade befindet. Vielmehr kennzeichnet a:visited dauerhaft alle Seiten, die der User in letzter Zeit besucht hat. Diese Informationen holt sich der Browser aus dem Verlauf(auch als Chronik oder History bezeichnet). Dabei ist es unerheblich, ob es sich um einen Link zu einer Unterseite desselben Webangebots oder um einen exteren Link handelt.

Sie müssen nicht zwingend alle Linkzustände definieren. In der Praxis beschränkt man sich häufig auf :hover

Es sollten daher zunächst allgemeine Vorgaben für Für alle Links gemacht werden. Dannach werden lediglich mit den Pseudoklassen die Änderungen beim Überfahren mit der Maus definiert.

NOTE: Bedenken Sie jedoch von vornherein, dass die Pseudoklasse :hoverauf Touchscreens naturgemäß nicht bzw. nicht wie erwartet funktioniert! viele Touchscreens versuchen zwar, den :hover zu emulieren, indem sie das erste Antippen als :hover und das zweite als Klick werten. Dies ist jedoch von zahlreichen zusätzlichen Faktoren abhängig und muss unbedingt getestet werden.

Verschiedene Zustände für andere Elemente definieren

Die Pseudoklassen :focus und :hovergelten nicht nur für Hyperlinks, Sie können sie jedem beliebigen Element zuweisen

Elemente automatisch durchzählen

CSS 3 ermöglichst es Ihnen, Elemente anhand ihrer Position im Dokument unterschiedlich zu gestalten. Dafür existieren u.a zwei Pseudoklassen:

  1. tag:nth-child(n)

    Mit dieser Pseudoklasse können Sie Elemente abhängig davon gestalten, das wievielte Kindelement eines Containers sie sind. Ein einfaches Beispiel

     <body>
       <p>Text</p>
       <p>Text</p>
       <p>Text3</p>
    </body>

    Um nun den dritten Absatz eine andere Textfarbe zu geben, notieren Sie:

    p:nth-child(3){color:red;}

    Der dritte Absatz(text3) wird nun rot gefärbt, weil er das dritte Kindelement im übergeordneten Container(Body) ist.

     <body>

      <div>
       <p>Text</p>
       <p>Text</p>
       <p>Text3</p>
      </div>

      <div>
       <p>Text</p>
       <p>Text</p>
       <p>Text6</p>
      </div>
    </body>

    Hier würden jedoch zwei Absätze(Text3,Text6) rot eingefärbt werden, da beide jeweils das dritte Kindelement in ihren jeweiligen Container(hier div) sind.

    Dafür steht N für eine beliebige Zahl(oder ein Schlüsselwort)

    Für das erste, letzte oder einzige Kindelement können Sie auch folgende Kurzschreibweise verwenden:

    1. :first-child

    2. :last-child

    3. :only-child

    Zebratabellen gestalten

    Neben Zahlen können Sie in den runden Klammern auch die Schlüsselwörter odd(=ungerade) und even(=gerade) Zahlen verwenden

    Bsp:
    tr:nth-child(odd){color:#fff;}
    tr:nth-child(even){color:#000;}

    NOTE: Achten Sie darauf, dass Sie immer das korrekte HTML-Element angeben. Für Zebratabellen müssen Sie die table rows auswählen(und nicht etwa td)

  2. tag:nth-of-type(n)

    Anders verhält sich die Pseudoklasse :nth-of-type(n) Damit wählen Sie ein Element abhängig davon aus, das wievielte seines Typs es im Dokument ist. Betrachten wir nochmals oberes Beispiel:

     <body>
       <p>Text</p>
       <p>Text</p>
       <p>Text3</p>
    </body>

    p:nth-child(3) oder p:nth-of-type(3) ist in diesem Fall egal, es würde in beiden Fällen, weil text3 sowohl das dritte Element im Body(nth-child(3)) ist, sowohl auch das dritte Element seines Typs(nth-of-type(3)) ist.

    Verändert man das Beispiel:

     <body>
       <h1>Text</h1>
       <p>Text</p>
       <p>Text2</p>
       <p>Text3</p>
    </body>

    Wird bei p:nth-of-type(3) wie gehabt Text3 ausgewählt, weil es das dritte tag seiner Art ist. Jedoch bei nth-child(3) text2, da es das dritte Kindelement im Body ist

    Auch hier gibt es die Kurzschreibweisen:

    1. :first-of-type

    2. :last-of-type

    3. :only-of-type

Sie können das Ganze noch weiter treiben, indem Sie in den unden Klammern anstatt einfacher Zahlen Ausdrücke nach folgenden Muster notieren:

:nth-child(bn+a)
:nth-of-type(bn + a)

Zum Beispiel: li:nth-child(3n-4)

Diese Ausdrücke liest man am Besten von hinten: Zunächst geben Sie mit a die Position des ersten Elments an, für das die Anwesungen gelten sollen, dann mit b die Position aller gewünschten elemente danach. Das Beispiel betrifft also zunächst das 4. Listenelement und danach jedes 3

Pseudoelemente

Pseudoelemente wurden in CSS 2 - genau wie Pseudoklassen - mit einem Doppelpunkt gebildet, in CSS 3 mit zwei Doppelpunkten Diese unterschiedliche Notation soll helfen, Pseudoelmente von Pseudoklassen zu unterscheiden.

CSS 2: p:first-letter
CSS 3: p::first-letter

Noch nicht alle Browser verstehen die Schreibweise mit zwei Doppelpunkten, Sie können daher auch Pseudoelemente weiterhin mit nur einem Doppelpunkt notieren.Ausnahme: Wenn das Pseudoelement erst in CSS 3 neu eingeführt wurde, müssen Sie es mit zwei Doppelpunkten schreiben()

Initialen und andere Typographische Effekte

Mit Pseudoelementnt lassen sich besondere typographische Effekte etwa für Absätze erzeugen:

  • p:first-line ⇒ formatiert die erste Zeile eines Textes anders

  • p:first-letter ⇒ formartiert das erste Zeichen des Textes als Initale

Pseudoelemente für automatisch erzeugten Inhalt

Sie können in CSS nahezu jedes beliebige Element automatisch Text oder Graphik anhängen. Diese Texte oder Bilder tauchen nirgends im HTML-Quelltext auf(und sind im Browser nicht anwählbar), sondern werden beim Laden der Seite automatisch generiert. Die Pseudoelemente hierfür lauten:

  • :before ⇒ Inhalt, der vor dem betreffenden Element eingefügt wird

  • :after ⇒ Inhalt, der nach dem betreffenden eingefügt wird

Beispiel Ihr HTML-Dokument enthält externe Links, die auf andere Webseiten führen. Diese Links möchten Sie optisch kennzeichen, indem Sie ihnen ein kleines Icon anhängen.

a:after{content: url(super.gif);}

Alternative kann man dazu auch text einfügen

a:after{content: " (externer Link)";}

Man kann auch beides kombinieren.

a:after{content: url(super.gif) " (externer Link)";}

before:funktioniert nach demselben Prinzip.
