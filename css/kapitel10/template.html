Kapitel 10 Größenangaben.

Angaben für Breite und Höhe

Es könne für jedes Element in Ihrem Dokument exakte Größenangaben mit den Attribute width (breite) und height (höhe) in Verbindung mit numerischen Werten machen. Es sind auch Prozentangaben sowie die Maßeinheiten em und rem möglich. Mit letztern können Sie "dynamische" Größenangaben machen, die sich zusammen mit der Schriftgröße ändern.

Beispiel:
.nav{
width: 30%
height:20em
}

Wenn Sie jedoch möchten, das sich die Höhe einese Elements automatisch dem Inhalt anpasst, sollten Sie in der Praxis das Attribut height in der Regel weglassen. Eine Ausnahme davon bildet beispielsweise der header von Websites, der in der Regel eine feste Höhe hat.

NOTE: Der Breite und der Höhe eines Elements werden nach dem W3C-Standard die Werte, die Sie für padding und border angeben, hinzugezählt

Beispiel

width: 150px;
height: 100px;
padding: 5px;
border: 1px solid #000

Die gesamtbreite des Elements im Browser beträgt dadurch:
150px + 5px(padding links) + 5px (padding rechts) + 1px (border links) + 1px (border rechts) = 162px

Die höhe beträgt entsprechend
100px + 5px + 5px + 1px +1px =112px

Übersatz

Wenn Sie eine feste Höhe angeben, kann es passieren, dass der Inhalt nicht mehr ins Element passt (man spricht hierbei von einem Übersatz). Standardmäßig ragt der Inhalt dann über das Element hinaus. In diesem Fall können Sie mit dem Attribut overflow bestimmen, was mit dem Inhalt geschehen soll:

Beispiel:
.nav{
width: 30%
height:20em
overflow: auto;
}

overflow:

visible (der Inhalt ragt gegebenfalls aus dem Element heraus, Standardwert)
hidden(der Inhalt wird ggt. abgeschnitten)
scroll(der Browser erzeugt Rollbalken für das Element, allerdings auch dann wenn gar keine benötigt werden.)
auto (der Browser erzeugt nur Rollbalken für das Element, wenn sie nötig sind)

Vor allem mit overflow:autokönnen Sie interessante Effekte erzeugen. Die so ausgezeichneten Elemente erhalten eigene Rollbalken, ähnlich einem iFrame-Fenster. Beachten Sie, dass der Effekt nur dann sichtbar ist, wenn das betreffende element eine feste Höhe hat und der Inahlt nicht mehr hineinpasst.

NOTE: Wenn Sie die Eigenschaft overflow:auto verwenden sollten Sie i.d.R zusätzliche auch die Eigenschaft:

-webkit-overflow-scrolling: touch;

angeben. diese gehört nicht zum W3C-Standard und ist eine proprietäre Eigenschaft für Webkit-basierte Browser. Sie sorgt dafür, dass das betreffende Element auch auf Webkit-basierten Mobilgeräten mit dem Finger gescrollt werden kann, Dabei wird am Ende auch der bekannte "Gummiband-Effekt erzeugt."

Manuelle Größenänderung.

Zusätzlich(oder alternativ dazu) können Sie dem Anwender auch die Möglichkeit geben, Elemente mit fester Breite oder höhe selbst zu vergrößern oder zu verkleinern. Dafür steht ihenen das Attribut resize zur Verfügung. Mögliche Werte sind:

resize:

both
vertical
horizontal
none(Standardeinstellung)

Die so ausgezeichneten Elemente bekommen an der rechten unteren Eche einen Anfasser, mit dem das Element größer und kleiner gezogen werden kann.

Mindest und Maximalgrößen

Analog zu width und height bieten CSS einige praktische Alternativen, so etwa die Mindestbreite min-width und die Mindesthöhe min-height. Damit kann ein Element eine bestimmte Breite bzw. Höhe nicht mehr unterscheiden, skaliert aber trotzdem mit , wenn der Inhalt länger wird. Analog dazu existieren auch die Eigenschaften max-height u. max-width
