Punktlisten

Punktelisten sind ungeordnete Listen. Als Aufzählungszeichen dient ein Spiegelpunkt. Folgender Code erzeugt eine solche Liste:

<ul>

<li>London</li>

<li>Berlin</li>

<li>Monaco</li>

<li>San Paulo</li>

</ul>

Die Liste wird mit dem Start-Tag<ul>(für unordered list) eingeleitet. Dieser Tag ist ein Blockelement und erzeugt einen Absatz. Die einzelnen Listenelemente selbst werden mit <li> der endtag </li> ist ebenfalls erforderlich. Die Listenelemente werden im Browser automatisch eingerückt.

Nummerierte Listen

Nummerierte Listen sind analog zu Aufzählungslisten aufgebaut, lediglich der Tag ändert sich in <ol>

Die liste wird im Browser automatisch durchnummeriert. die voreingestellte Nummerierung erfolgt mit arabischen Ziffern(1,2,3,4). Sie kann jedoch mit dem type-Attribut geändert werden:

<ol type="A" = A,B,C,D,E ......

<ol type="a" = a,b,c,d,e ......

<ol type="I" = I, II, III, IV, V..........

<ol type="i" = i, ii , iii, iv, v.........

Mit dem Attribute start können Sie den Anfoangswert Anfangswert für die Listennummerierung ändern:

<ol start="6">

Sie können auch die Attribute type und start auch kombinieren:

<ol start="6" type="a">

Die Liste beginnt so mit dem Buchstaben f.

Sie können die Nummerierung auch innerhalb in der Liste wechseln. Notieren Sie dazu das Attribut value innerhalb des betreffenden li-Tags

<li>London</li>

<li value="22">Berlin</li>

<li>Köln</li>

<li>Münster</li>

Note: Verwehseln Sie nicht die Attribue start und value. Das Attribut start gehört in den ol-tag und ändert den Anfangswert der Aufzählung, value gehört in den li-tag und ändert den Wert innerhalb einer Liste.

Listen und Listenelemete können mit CSS umfangreich formatiert werden, sodass es z.B möglich ist, damit Navigationsmenüs zu erstellen.

Definitionslisten

definitionslisten gehen eine Stufe tiefer in der Hierarchie: Auf der ersten Ebene können Sie Hauptkategorien (Schlagwörter)aufzählen, denen Sie dann auf der zweiten Ebene Unterkategorien zuordnen können. Definitionslisten verzichten auf Aufzählungszeichen, die Definitionen werden im Browser lediglich eingerückt dargestellt.

Mit <dl> leiten Sie eine Definitionsliste ein. Danach folgt mit dem Tag<dt> der este definition term, also die erste hauptkategorie. Nun folgt eine beliebige Anzahl an definition data <dd>,bevor das nächste <dt> genannt wird.

<dl>

 <dt>Kino</dt>

   <dd>Hulk</dd>

   <dd>Mulan</dd>

   <dd>Nintendo</dd>

 <dt>DVD</dt>

   <dd>Revolver</dd>

   <dd>Peter</dd>

   <dd>Ghots Nachricht von Sam</dd>

</dl>
Verschachtelte Listen

Alternativ zu Definitionslisten können Sie auch Punkt- oder nummerierte Listen ineinander verschachteln. Dazu erstellen Sie innerhalb eines li-Tags einfach eine komplette neue Liste.

Ausklapp-Texte

Aufklapp-Texte können in einigen Fällen evtl. als Alternative zu Definitions- oder erschachtelten Listen verwendet werden.

Diese Elemente mussten in der Vergangenheit aufwendig mit CSS und JavaScript selbst programmiert werden. HTML5 bietet diese Funktionalität nun vorgefertigt.

Ein Ausklapp-Text besteht aus zwei Teilen: dem umgebenden Tag <details> und dem darin enthaltenen Tag <summary>. Der <summary>-Tag enthält die sichtbare Überschrift. Danach folgt der - zunächst unsichtbare - Inhalt (der mit beliebigen weiteren Tags formatiert werden kann)

<details>

 <summary>Kino</summary>

   <dd>Hulk</dd>

   <dd>Mulan</dd>

</details>

Standardmäßig ausgeklappter Text

Sie können aber auch festlegen, dass der Inhalt bereits von vornherein ausgeklappt erscheint. Fügen Sie dazu einfach dem einleitenden details-Tag das attribute open hinzu
