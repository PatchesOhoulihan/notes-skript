Kapitel 3 Textformatierung

Trivia

  • Besonderheiten bei der Textformatierung mit HTML

    • Tabulatoren und Zeilenumbrüche um Code werden im Browser wie Leerzeichen behandelt.

    • Mehrere aufeinanderfolgende Leerzeichen, Tabulatoren und Zeilenumbrüche werden im Browser immer nur als ein einziges Leerzeichen angezeigt.

    • Deshalb werden diese Zeichen zusammenfassend als white space bezeichnet.

Blockelemente

Absätze

  • Die einfachste und effektivste Methode zur Strukturierung von Fließtexten ist die Verwendung von Absätzen.

    <p> Lorem ipsum dolor sit amet. </p>

Überschriften

  • Die sechs h-Tags können für hierarchische Überschriften benutzt werden. Überschriften werden als eigener Absatz behandelt.

    <h1-h6> Lorem ipsum. </h1-h6>

Trennlinien

  • Ein weiteres Element zur Abgrenzung von thematisch nicht zusammenhängenden Textbereichen ist die Trennlinie, die einen eigenen Absatz erzeugt und mit dem Tag

    Syntax: <hr>

Weitere Blockelemente zur Strukturierung von Dokumenten

Tag Beschreibung
<article> Neu in HTML5. Unabhängiger Abschnitt.(z.b ein Blog-Beitrag.)
<aside> Neu in HTML5. Inhalt, der zum Thema passt, aber nicht direkt dazugehärt, Randbemerkung, Marginalie. Wird auch gerne für Seitenleisten(sidebars) verwendet
<blockquote> Langes Zitat. Bildet im Gegensatz zu <q> einen eigenen Absatz.
<div> Vielseitig einsetzbar zur Kennzeichnung eines Abschnitts oder Bereichs. Sollte aber nur verwendet werden, wenn keines der anderen Elemente(article, section, aside, main, header, footer usw.)passt
<footer> Neu in HTML5. Fußzeile eines Abschnitts oder einer Seite.
<header> Neu in HTML5. Einleitung/Kopfbeeich eines Textes oder einer Seite(nicht mit head verwechseln.)
<main> Neu in HTML5. Kennzeichnet den Haupt-Inhaltsbereich.
<nav> Neu in HTML5. Kennzeichent die darin enthaltenen elemente(i.d.R. Links) als Navigationselemente.
<section> Neu in HTML5. Abschnitt;Sektion eines Textes. Kann z.B. zur weiteren Unterteilung von main oder article verwendet werden.

Viele dieser Tags - vor allem die in HTML5 neu hinzugekommenen - erzeugen im Browser jedoch keine sichtbare Veränderung! Sie sind daher erst im Zusammenspiel mit CSS wirklich interessant.

Die meisten der oben genannten Tags sind optional, und für ihre Verwendung gibt es kaum feste Regeln. Es ist also auch Ihrer Kreativität überlassen, wann Sie welchen Tag für sinnvoll halten. Lassen Sie jedoch Ihre Dokumente regelmäßig validieren, um zu sehen, ob Sie einen Tag evtl. doch einmal im falschen Zusammenhang verwendet haben.

Inline-Elemente

Fettschrift

Zur Auszeichnung stark betonter Textbereiche nutzen Sie den Tag <strong>fetter text </strong>

Syntax: dies ist ein <strong>fetter text </strong>

Alternativ dazu bietet HTML5 auch den Tag <b> für bold. Dieser sollte jodoch nur in Ausnahmefällen benutzt werden, wenn Textbereiche vom umgebenden Text abgegrenzt werden sollen, ohne sie dadurch wichtiger erscheinen zu lassen oder stärker zu betonen. Beispiele dafür wären etwa die Namen der Sprecher in Interviews oder Theaterstücken Diese werden optisch hervorgehoben, sind dadurch aber nicht wichtiger.

Kursivschrift

Zur Auszeichnung betonter Textbereiche nutzen sie den Tag <em></em>.Dieser wird in allen gängigen Browsern kursiv angezeigt.

Dies ist ein kursiver Text.

Weitere Tags zur Textauszeichnung

<abbr> Kenzeichnet einen Textbereich als Abkürzung
<address> Kennzeichnet einen Textbereich als Kontaktinformation, i.d.R des Autors eines Dokuments oder Artikels.
<cite> Kennzeichnet einen Textbereich als Titel eines Werkes, das im Text zitiert wird
<code> Kennzeichnet einen Textbereich als Quelltext, z.B. zur Visualisierung von PHP-Code auf einer Webseite.
<del> Kennzeichnet einen Textbereich als gelöscht. Der Text wird in der Regel durchgestrichen angezeigt. Wrid oft zusammen mit <ins>+
<ins> Kennzeichnet einene Textbereich als nachträglich eingefügt. Wrid oft zusammen mit <del> der Text wird in der Regel unterstrichen angezeigt. Dies sollten sie auf jedenfall mit css ändern, da der Text sonst als Link missverstanden werden kann!
<mark> Neu in HTML5. Markiert einen Textbereich wie mit einem Edding
<q> Kurzes Zitat im Fließtext. Zur Kennzeichnung eines ganzen Absatzes als Zitat nutzen Sie stattdessen das Block-Element &
<s> Kennzeichnet einen Textbereich als falsch(engl. strikethrough) bzw. nicht mehr korrekt. Ähnlich, aber nicht zu verwechseln mit <del>. Der Text wird in der Regel durchgestrichen durchgestrichen angezeigt
<sub> Tiefgestellter Text(Subscript)
<sup> Hochgestellter Text(Superscript)
