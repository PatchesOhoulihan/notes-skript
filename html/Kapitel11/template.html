Meta-Tags ermöglichen es Ihnen, zusätzliche Informationen über Ihr Dokument in Ihre webseiten einzufügen. Diese Infromationen können z.B. von Suchmaschinen oder Webbrowsern ausgelesen werden, sind aber für die Besucher Ihrer website unsichtbar. Sie werden im Kopfbereich <head> der HTML-Seite notiert.

Für Meta-Tags gab es nie einen allgemeinverbindlichen Standard, was im Laufe der Zeit zu einem regelrechten Wildwuchs an Meta-Tags im web geführt hat.

Allgemeine Syntax

Es gibt zwei verschiedene Schreibweisen, Meta-Tags zu notieren. Welche Alternative Sie benutzen müssen, ist vom jeweiligen Meta-Tag abhängig.

  • <meta name="..." content="..." />

  • <meta http-equiv="..." content="..." />

Das erste Attribut bestimmt, um welchen Meta-TAg es sich handelt. In vielen Fällen wird dafür das Attribut name verwendet.

Im zweiten Attribut content geben Sie den Wert bzw. Inhalt an, den Sie diesem Meta-Tag zuweisen wollen.

Eine Ausnahme davon bildet lediglich der Meta-Tag zur verwendeten Zeichenkodierung Dieser kann set HTML5 als <meta charset="utf-8"/> notiert werden.

Angaben für Suchmaschinen

1. Beschreibung des Seiteninhalts

Die Angaben in diesem Meta-Tag werden in den Trefferlisten mancher Suchmaschinen angezeigt, wenn Ihr eintrag gefunden wurde. Er sollte eine kurze, prägnante Beschreibung des Seiteninhaltes und nicht mehr als 256 Zeichen oder 25 Wörter beinhalte.

<meta name="description" content="Beschreibung des Inhalts"/>

2. Autor

Hiermit kann der Author der Seite genannt werden

<meta name="Author" content="Name des Verantwortlichen"/>

3. Suchbegriffe(Keywords)

Dieser Meta-Tag wird - aufgrund häufigen Missbrauchs heute von KEINER Suchmaschine mehr beachtet. Er sollte die Suchbegriffe enthalten, unter denen Ihre Seite gefunden werden soll.

<meta name="keywords" content="Stichworte"/>

Angaben für Browser

1. Zeichenkodierung

Sie sollten immer angeben mit welcher Zeichkodierung Sie Ihr Dokument gespeichert haben.

<meta charset="utf-8"/>

2. Weiterleitung

Diesen Meta-Tag können sie verwenden, wenn Sie die Besucher Iherer Site nach einer gewissen Zeitspanne automatisch auf eine andere Seite weiterleiten wollen.

< meta http-equiv="refresh" content="5 ;url=http="http://www.google.de"

die Zahlenangabe im Attribut content bestimmt die Anzeigedauer in Sekunden, bevor die neue Datei aufgerufen wird

3. Größe des Viewport

ein im Zusammenhang mit Responsive Webdesing mittlerweile unerlässlicher Meta-Tag ist die Angabe der Größe bzw. Skalierung des Viewports.

Normalerweise verkleinern mobilegeräte eine Website, sodass sie(nahezu) vollständig ins Anzeigefenster passt Dies ist beim Responsiv Webdesign jedoch nicht gewünscht - stattdessen passt hier der Webdesigner das Lauyout der Webiste dynamisch an die Größe des Viewports an.

<meta name="viewport" content="width=device-width, inintial-scale=1.0"/>

Im Attribute contentkönnen Sie - durch Kommas getrennt - gleich mehrere Angaben machen:

  • width gibt an, auf welcher Größe der Viewport skaliert werden soll. Auch numerische Werte möglich. Um aber die Skalierung abzuschalten an die tatsächlichen Bilschirmgröße des jeweiligen Gerätes anzupassen, In der Praxis sollte man immer device-width notieren.

  • initial-scale sollten Sie zusätzlich anoch angeben, das die Seite anfänglich auch nicht gezoomt werden soll. 1.0 entspricht 100%

  • user-scalable=no sorgt dafür das manuelle Zoomen mit zwei zwei Fingern komplett deaktiviert ist.

Dieser Meta-Tag ist natürlich erst im Zusammenhang mit CSS wirklich sinnvoll, wo Sie für die einzelnen Elemente auf Ihrer Website exakte Größen- und Postionsangaben machen können.

Diese Angabe wirkt nur auf Mobilgeräten aus. Auf dem Desktop-Browser hat sie keinerlei Einfluss

4. Spezifische Angaben für den Internet Explorer

Der IE verfügt ab der Version 8 über zwei Dartsellungsmodi, den Standardmodus und den sogenannten Kompatibilitätsmodus.

Im Kompatibilitätsmodus emuliert der IE 8 absichtlich Fehldarstellungen und Unzulänglichkeiten älterer IE-Versionen

Welchen Modus der Browser im Einzelfall benutzt, bleibt weitgehend dem Anwender überlassen. Wir brauchen hier also eine Möglichkeit, den IE 8 in den Standardmodus zu "zwingen"

<!--[if IE]> <meta http-equiv="X-UA-Compatible" content="IE=edge"\><![endif]-->

Mit diesem Meta-Tag sollte der IE die neue edge rendering engine zuverwenden.

Es muss darauf geachtet werden, diesen Meta-Tag VOR allen verknüpften CSS und Javascript-Dateien zu platzieren.

Diesen Meta-Tag sollten sie von nun an in den head jeder ihrer Webseiten verwenden.

Da dieser Meta-Tag nach der HTML-5-Spezifikation jedoch nicht valide ist, sind hier sog. Conditional Comments nötig. Dabei handelt es sich ebenfalls um eine von Microsoft entwickelte Technik: Für andere Browser und Validatoren sieht die Ausgabe wie ein normaler Kommentar aus und wird ignoriert. Der IE erkent den Tag jedoch aufgrund von [if IE]
