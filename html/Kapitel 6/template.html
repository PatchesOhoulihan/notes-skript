Hyperlinks

Einen Hyperlink definieren

Grundlagen

Sie können sowohl einen Textbereich als auch eine Grafik als Hyperlink definieren

<a>Textlink<</a> <a><img src="buttongrafik.gif"/></a>

Der a-Tag ist normalerweise ein Inline-Element. Seit HTML5 kann er aber um Blockelemente wie <h1> oder <p> herum gesetzt werden, um diese zu verlinken.

Momentan passiert beim Klicken auf den Hyperlink allerdings noch herzlich wenig. Schließlich müssen Sie erst noch definieren, wohin Sie der Link bringen soll Dafür gibt es das Attribut href

<a>Textlink</a>

Das Element innerhalb des Anchor-Tagpaares wird nun als Hyperlink dargestellt, daher:

  1. Text erscheint standardmäßig unterstrichen und in blauer Farbe auf dem Bildschirm

  2. Bilder erhalten in manchen Browsern eine blaue Kontur(beides lässt sich natürlich mit CSS ändern.)

  3. Bewegt der Anwender den Mauszeiger über das Element, verwandelt sich dieser in einen Finger.

Das Prinzip der Referenzierung im href-Attribut ist dasselbe, das Sie auch bei der Referenzierung von Grafiken verwenden. Sie können sowohl relative als auch absolute Pfadangaben verwenden. Dabei gilt grundsätzlich die Unix-syntax für den Verzeichniswechsel, unabhängig vom Betriebssystem, auf dem der Webserver läuft.

Relative Pfadangaben

Wenn Sie die einzelnen Seiten Ihrer Website untereinander verlinken wollen(man spricht hier auch von internen Links), benutzen Sie normalerweise - wie bei der Referenzierung von Bildern - relative Pfadangaben, ausgehend vom Speicherort des Quelldokuments. Im einfachsten Fall verweisen Sie auf ein weiteres HTML-Dokument, das sich im selben Ordner wie das Quelldokument befindet:

  1. Beispiel 1:

    <a href="index.html">Zurück zur Startseite</a>

  2. Beispiel 2:

    Befindet sich das Zieldokument in einem Unterordener(z.B. im Ordner produkte), sieht der Link folgendermaßen aus:

    <a href="produkte/digicam.html">Digikam</a>

  3. Beispiel 3:

    <a href="../index.html">start</a>

    Für Links auf Dokumente, die sich eine Verzeichnisebene über dem Quelldokument befinden

Als allgemeine Regel für relative Pfadangaben gilt also:

  • um eine Ebene nach unten zu gelangen, geben sie einfach den Ordnernamen an, gefolgt von einem Schrägstrich /.

  • mit ../ springen Sie in der Ordnerhierachie nach eine Ebene nach oben. mit ../../ zwei Ebenen usw.

Absolute Pfadangaben

Interne Links

In manchen Situationen(z.B beim Einsatz von Skriptsprachen oder lokalen Suchmaschinen kann es nötig sein, absolute Pfadangaben zu verwenden.)

Absolute Pfadangaben funktionieren nur, wenn Sie über den Webserver aufgerufen werden, nicht jedoch, wenn Sie die Dateien lokal von der Festplatte Ihres Computers betrachten.

Wie unter Unix, üblich beginnen absolute Pfade stets mit dem nach vorne gerichteten Schrägstrich/(forward slash). Damit befinden Sie sich im Root-Verzeichnis des Webservers und rufen von dort einfach den vollständigen Pfad der Zieldatei auf.

Bsp. <a href="/index.html">absoluterPfad von Root</a>

Externe Links

Links die sich auf irgendeinem Webserver der Erde befinden, referenzieren. Dazu müssen Sie nur die richtige URL angeben, die eindeutige Adresse einer Datei im Internet.

Bsp. <a href="http://www.google.de">externer Link</a>

Wichtig dabei ist, dass Sie das Protokoll angeben(meist http:// odr - bei verschlüsselten Seiten https://). Beachten Sie, dass in diesem Beispiel kein Dateiname angegeben ist. Webserver sind im allgemeinen so konfiguriert, dass sie im Stammverzeichnis des angegebenen Webservers nach einer Datei index.html suchen, sofern nicht explizit ein Dateiname angegeben wurde.

Wenn Sie jedoch auf eine bestimme Unterseite eines Fremden Webangebots verlinken möchten, benötigen Sie natürlich absolute Pfadangaben: den Namen der Zieldatei inklusive aller Verzeichnisnamen:

Bsp. <a href="http://www.google.de/deinemama.html">externer Link</a>

Zielfenster definieren

Standardmäßig werden Links im selben Browserfenster geöffnet, von dem aus sie aufgerufen wurden Sie können jedoch mit Hilfe des Attributes target="_blank" vorgeben, dass die Zielseite in einem neuen, leeren Browserfenster bzw. Tab geöffent wird(beachten Sie den Unterstrich vor blank). Die ursprüngliche Seite bleibt in diesem Fall hinter dem neuen Fenster erhalten, sodass der Anwender jederzeit wieder zu ihr zurückkehren kann.

Dies ist vor allem dann sinnvoll, wenn Sie externe Links auf andere Websites verwenden, dabei aber sicherstellen wollen, dass der Besucher Ihr Webangebot nicht dauerhaft verlässt:

Bsp. <a href="http://www.google.de/" target="_blank">externer Link</a>

Es gibt noch andere Werte außer _blank, die Sie im Attribut target verwenden können. Diese sind jedoch nur sinnvoll im Zusammenhang mit sog. Frames

Downloads

Sie sind beim Verlinken von Dateien natürlich nicht auf HTML-Dokumente beschränkt, sondern Sie können - mit jeder der o.g Methoden egal ob extern oder internen links - theoretisch auf jeden beliebigen Datentyp verweisen und dadurch z.b. Dateien zum Download anbieten.

Bsp. <a href="http://www.google.de/downloads.zip" target="_blank">externer Link</a>

Bsp. <a href="downloads/proc.exe" target="_blank">interner link Link</a>

Dabei ist es allerdings den Browser- und den Systemeinstellungen des Clientrechners überlassen, ob die Datei im Browserfenster angezeigt oder auf die lokale Festplatte zur weiteren Bearbeitung heruntergeladen wird(viele Browser überlassen auch dem Benutzer die Wahl, was mit der Datei geschehen soll). Zwar werden ZIP-Dateien standardmäßig heruntergeladen und JPEG-Bilder im Browser angezeigt. Bei PDF-Dateien hingegen gibt es beide Möglichkeiten, abhängig davon, ob der Benutzer ein Browser-Plugin zur Anzeige von PDF-Dateien installiert hat oder nicht.

Der mailto-link

Wenn Sie den Besuchern ihrer Website die Möglichkeiten geben wollen, mit Ihnen per Email E-Mail in Kontakt zu treten, können Sie einen speziellen Verweis definieren, der das voreingestellte E-Mail-Programm des Anwenders öffnet und bereits die angebene E-mail-Adresse als Empfänger einträgt:

Bsp. <a href="http:mailto:webmaster@xyz.com" target="_blank">mailto</a>

Beachten Sie, dass hier nach dem Doppelpunkt keine Schrägstriche folgen.

Auch die Betreffzeile(subject) und der Email-text selbst können vorgegeben werden.

Bsp. <a href="http:mailto:webmaster@xyz.com?subject=betreffzeile&body=bodytext target="_blank">mailto</a>

Beachten Sie jedoch, dass der Anwender in diesem Fall über einen fertig konfigurierten Email-Client verfügen muss. Ist dies nicht der Fall, erhält er eine Fehlermeldung und hat keine Möglichkeit, mit Ihnen in Kontakt zu treten. Zudem öffnet der mailto-link Tür und Tor für Spam. Vermeiden Sie ihn deshalb in der Praxis und benutzen stattdessen ein in Ihre Website integriertes Kontaktformular

Links zu bestimmten Stellen einer Webseite

Sie können mit dem a-Tag nicht nur links auf andere Seiten definieren, sondern bei Bedarf auch zu bestimmten Textstellen auf derselben oder einer anderen Seite springen. Dies ist z.B dannn sinnvoll, wenn Sie eine Liste häufig gestellter Fragen(faq) auf ihrer Website anbieten: Beim Klick auf eine Frag springt der Browser direkt zu einer Stelle auf der Seite, an der sich die entsprechende Antwort befindet. Auch die oft verwendeten Links zurück zum Seitenanfang werden auf diese Weise realisiert.

Ein solcher Link besteht aus zwei Teilen:

  1. einer(für Benutzer unsichtbaren) Markierung an der Stelle, an die gesprungen werden soll

  2. einem(sichtbaren) Link, der zu dieser Markierung führt.

Die Zielmarkierung wird mit dem Attribut id realisiert, sas Sie jedem beliebigen Tag zuweisen können

<p id="meineId"></p> wnn zu einem bestimmten Absatz gesprungen werden sol.

Ober <h1 id="meineId"></h1> wenn zu einer bestimmten Überschrift gesprungen werden soll.

Als Id benutzen Sie einen frei wählbaren Begriff. dieser darf jedoch keine leer- oder Sonderzeichen enthalten, und Sie müssen auf Groß- und Kleinschreibung achten. Beachten Sie, dass der Name eindeutig sein muss , d.h er darf nur einmal im Dokument vorkommen.

IDs sind äußerst flexibel einsetzbar und können auch in anderen Zusammenhängen sehr nützlich sein. so werden sie z.B auch benutzt, um HTML-Elemente mit CSS zu gestalten oder mit Javascript zu modifizieren.

Links zu einer Textstelle auf einer anderen Seite

Nehmen wir an, Sie wollen von der Seite fragen.html zu einer bestimmten Textstelle auf Seite antworten.html verwwisen. Dazu müssen Sie an der gewünschten Stelle auf der Seite antworten.html einen Anker setzen.

Ober<p id="meineId">Die erste antwort</p>

Auf der anderen Seite setzen Sie den Link wie folgt:

Ober<a id="SeiteMitMeineId.html#meineID">link zu meine Id</a>

Der Name der Zielmarkierung wird also einfach a die URL angehängt. Wichtig dabei zwischen url und Raute# darf kein Leerzeichen stehen!

Imagemaps

Stellen Sie sich vor, Ihr Auftrag ist es, auf der Website eines Unternehmen ein Gruppenfoto zu platzieren, auf dem alle Kundenberater abgebildet sind. Jeder der Kundenberater soll einzeln anklickbar sein

Mit unseren bisherigen Methode, einen Hyperlink um den entsprechenden img-Tag zu legen, kommen wir hier nicht weiter, da damit nur das komplette Foto verlinkt werden kann.

Eine exaktere Methode bieten sogenannte Imagemaps. Mit ihrer Hilfe können Sie mehreren -auch unförmigen - Bereichen innerhalb eines Bildes verschiedene Hyperlinks zuweisen, indem Sie eine (für den Anwender unsichtbare) "Landkarte" über das Bild legen, die dieverschiedenen aktiven Bereiche kennzeichnet. Für die Definition dieser Bereiche in HTML müssen Sie die genauen Koordinaten innerhalb eines Bildes angeben, in denen der Verweis liegt

Definition

die definition einer Imagmap besteht aus einer Kombination zweier zusammengehöriger Tag: <map> und <area>. Die Imagemap kann an jeder Stelle im Body verwendet werden.

<map name="gruppenbild"></map>

Innerhalb des map-Tagpaares definieren Sie die einzelnen verweissenstiven Flächen Ihrer Grafik mit Hilfe des Standalone-Tags area. Für jeden Bereich, der anklickbar sein soll, benötigen Sie eine weitere area.

Innerhalb des map-Tagpaares definieren Sie die einzelnen verweissensitiven Flächen Ihrer Grafik mit Hilfe des Standalone-Tags area. Für jeden Bereich, der anklickbar sein soll, benötigen Sie eine weitere area

<map name="gruppenbild">
<area />
<area />
</map>

Bei der Definition der areas haben Sie die Wahl zwischen drei Grundformen, die Sie mit dem dem Pflichtattribut shape vorgeben:

  1. <area shape=""/>(Viereck)

  2. <area shape=""/>(Kreis)

  3. <area shape=""/>(Polygon, beliebiges Vieleck)

Mit dem Attibut coords geben Sie die Koordination der verweissenitiven Flächen an. Die Angaben verstehen sich als absolute Pixelwerte innerhalb der Grafik und werden durch Kommas getrennt. Beim Ausmessen der Koordinaten kann Ihnen ein Bildbearbeitungsprogramm wie Photoshop behilflich sein.

Viereck

Das Viereck(shape="rect")definieren Sie mit vier Koordinaten.

  • x1=linke obere Ecke, Anzahl Pixel von links.

  • y1=linke obere Ecke, Anzahl Pixel von oben.

  • x2=rechte untere Ecke, Anzahl Pixel von oben.

  • y2=rechte untere Ecke, Anzahl Pixel von von oben.

Beispiel

<area shape="rect" coords="10,12,120,80" />

Kreis

Den Kreis(shape="circle") definieren Sie mit den drei Koordinaten für x,y,r

  • x= mittelpunkt des Kreises, Anzahl der Pixel von Links.

  • y=Mittelpunkt des Kreises, Anzahl der Pixel von oben.

  • r=Radius des Kreises in Pixeln

<area shape="circle" coords="10,12,120" />

Polygon

Das Polygon (shape="poly") definieren Sie mit der entsprechenden Anzahl Koordinaten für x1,y1,x2,y2.. xn,yn. Sie benötigen also für jede Ecke ein Koordinaten x/y:

  • x= Anzahl Pixel einer Ecke von links.

  • y= Anzahl Pixel einer Ecke von Oben.

Sie können beliebig viele Ecken definieren. Die Linie von der letzten zurück zur ersten Ecke, die das Polygon schließt, müssen Sie sich hinzudenken.

<area shape="poly" coords="10,12,120,23,45,67" />

Bereiche verlinken

Abschließend geben Sie in jeder area mit dem Attribut href an, auf welche Datei verwiesen werden soll.

die Beziehung zwischen Grafik und Imagemap stellen Sie über ein Attribut im img-Tag her:

<img shape="circle" coords="10,12,120" usemap="#gruppenbild"/>

Das Attribut usemap legt die an anderer Stelle definierte Imagemap gleichsam wie eine Schablone über das Bild. Als Wert geben Sie in Anführungszeichen den Namen Ihrer Imagemap ein. Vergessen Sie nicht die Raute, die hier die hier wiederum kennzeichnet, das es sich um eine selbtdefinierten Namen handelt.

Im Browser ist nun zwar keine optische Veränderung an der Grafik sichtbar. Bewegen Sie jedoch den Mauszeiger darüber, erkennen Sie, dass die einzelnen Flächen anklickbar sind.

Fehlt nur noch der alt-Text, der laut W3C-Standard - wie für Bilder - auch für den area-Tag ein Pflichtattribut ist.

<area shape="circle" coords="10,12,120" href="schniepo.html alt="Herr Schmidt"/>

Wenn Sie möchten, können Sie nun noch Tooltips für die einzelnen Bereiche vergeben, um dem Anwender beim Überfahren mit der anzuzeigen, wohin der Link ihn führt. Notieren Sie dazu in jedem area-Tag das Attibut title und vergeben Sie eine aussagekräftigen Text.
