Kapitel 3 Verschachtelte Selektoren.

Verschachteln von kombinierten Selektoren

LESS bietet die Möglichkeit, kombinierte Selektoren wie den Descendant oder den Child Selektor - analog zu HTML - zu verschachteln, im Grunde also das DOM direkt abzubilden.

Beispiele

CSS LESS
nav ul {} nav{
  ul{}
}
nav ul li{} nav{
  ul{
    li{}
  }
}
nav > ul{} nav {
   > ul{}
}
nav + ul{} nav {
   + ul{}
}

Dies macht den Code verständlicher, bei umfangreichen Vorgaben vor allem aber auch kürzer. Wenn Sie in CSS z.B schreiben.

Wie Sie sehen, ist hier jedoch eine saubere Einrückung zwingende Voraussetzung, um den Code noch zu verstehen.

Verschachteln von Pseudoklassen und Pseudoelementen

CSS

LESS

a{//Eigenschaften}
a:hover{//Eigenschaften}

a{
   ∧:hover{//Eigenschaften}
}

Achten Sie auf das ∧ Zeichen! Dies ist die zwingende Vorraussetzung dafür, dass die Pseudoformate ohne Leerzeichen an den Selektor angehängt werden. Ohne das ∧-Zeichen würde der Selektor a:hover in der kompilierten CSS-Datei so aussehen:

a :hover
