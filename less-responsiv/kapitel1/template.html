Die Limitationen von CSS

Durch die immer umfangreicheren Funktionen von CSS 3 werden auch einige Limitationen immer deutlicher, die die Arbeit mit CSS erschweren und unnötig umständlich machen. Solche Limitationen sind:

  1. In CSS sind keine Variablen möglich

    Stellen Sie sich vor, Sie haben für Ihre Website eine bestimmte Akzentfarbe, z.B. #f60. Wenn Sie diese Farbe verchiedenen Elementen auf Ihrer Seite - etwa Textbereichen oder Konturen - zuweisen möchten, müssen Sie sie jedesmal erneut notieren. Sie müssen bei einer Änderung diese Farbe an allen Stellen in Ihrem CSS-dokument einzeln ändern! Wäre es nicht schön, wenn Sie die Farbe nur einmal an einer zentralen Stelle in Ihrem Dokument als Variable definieren könnten? Bei einer Änderung der Variablen würde die Farbe in Ihrem gesamten CSS-Dokument automatisch angepasst werden

  2. CSS-Selektoren lassen sich nicht verschachteln

    Bei dem Descendant Selector müssen Sie jedesmal den kompletten Pfad notieren, z.B

    #header{irgendwas}

    #header nav{irgendwas}

    #header nav ul{irgendwas}

    #header nav ul li{irgendwas}

    #header nav ul li a{irgendwas}

    #header nav ul li a:hover{irgendwas}

    Hier würde es enorm viel Tipparbeit sparen, wenn Sie die Selektoren - analog zu HTML - verschachteln im Grunde also das DOM direkt abbilden könnten

  3. Klassen lassen sich nicht andere Selektoren als Eigenschaft zuweisen

    Es kommt häufig vor, dass Sie verschiedene Elementen auf Ihrer Seite immer wieder dieselben Eigenschaften zuweisen möchten. Wäre es nicht enorm praktisch, wenn Sie diese Eigenschaften zentral in einer Klasse speichern und diese Klasse dann ihrerseits den verschiedenen Elementen zuweisen zu könnten

    Dies alles ist möglich. Es gibt dafür schon seit geraumer Zeit gleich mehrere konkurrierende Technologien, sogenannte CSS Preprocessoren. Die beiden bekanntesten sind Less und SCSS. Die beiden unterscheiden sich mittlerweile fast nur noch in Syntaktischen Details.

Was ist LESS

LESS-Dateien können genauso wie CSS-Dateien in jedem beliebigen Editor erzeugt werden. Die Syntax von LESS erweitert die Syntax von CSS um die oben genannten Funktionen. So können Sie in LESS folgendes schreiben, um wie in Beispiel 1 eine Variable für eine globale Farbe zu definieren.

@meine_farbe: #f60

h1{color: @meine_farbe}

header{border: 1px solid @meine_farbe}

In Zukunft müssen Sie nur noch den Wert der Variablen @meine_farbe ändern, um diese Farbe in Ihrem gesamten Dokument anzupassen.

#header{
  nav{
    ul{
    }
  }
}

Die Verschachtelung der Selektoren entspricht exakt dem HTML-DOM und ist dadurch verständlicher, vor allem aber auch kürzer!

.box{
  border: 1px solid #666
  margin-bottom: 1rem;
  background-color:silver;}

div.loginbox{
  
.box
}

Das div-Element übernehmen dadruch automatisch sämtliche Eigenschaften der Klasse box. Auch hier müssen die Eigenschaften von nun an nur noch einmal zentral in der Klasse box geändert werden.

Durch diese Syntax sind LESS-Dokumente jedoch keine gültigen CSS-Dokumente mehr und würden von keinen Browser verstanden werden. Deshalb müssen LESS-Dokumente zunächst in "normale" CSS-Dokumente rückübersetzt(kompilert) werden, bevor sie in ein HTML-Dokument eingebunden werden können.

Wie funktioniert LESS

Zu diesem Zweck gibt es eine ganze Reihe von Programmen(Compiler) mit unterschiedlichen Funktionsumfang Eine umfangreiche Liste verschiedener Betriebssysteme finden Sie unter Less Compiler. In diesem Buch wird Koala für Mac, Windows und Linux verwendet.

NOTE: In einigen Editoren ist ein LESS-Compiler bereits integriert bzw. in Adobe Brackets. In diesem Fall werden externe Compiler wie Koala natürlich nicht benötigt.

Es folgt eine beschreibung von Koala, siehe Script Seite 16.

NOTE: Unter Komprimieren versteht man in diesem Fall das Entfernen sämtlicher unnötiger Leerzeichen und Zeilenumbrüche aus der Datei. Der Quelltext wird dadurch nahezu unlesbar, was aber hier natürlich keine Rolle spielt, da ja nicht Sie den Code lesen müssen, sondern nur der Browser, und dem sind Leerzeichen und Zeilenumbrüche egal.

Solange das Programm geöffnet ist, werden sämtliche Ordner in der Liste überwacht, d.h jedesmal, wenn sie eine LESS-Datei in einem beliebigen Editor bearbeiten und speichern, wird diese automatisch kompiliert, und es wird eine entsprechende CSS-Datei im selben Ordner erzeugt. Gleichnamige CSS-Dateien werden dabei überschrieben.

Geht etwas schief, wird vom Compiler eine Fehlermeldung angezeigt.

In der resultierenden CSS-Datei gehen natürlich alle Vorteile von LESS wieder verloren, doh das stört uns nicht, da diese Datei nur für den Browser gedacht ist Wir arbeiten ja weiterhin in der LESS-Datei.

Dadurch, dass bei jedem Speichern der LESS-Dateien die CSS-Dateien automatisch aktualisiert werden, verlieren Sie auch nicht die Möglichkeit Ihre Änderungen sofort im Browser zu testen.

NOTE: Zusammenfassend bedeutet dies:

  1. Alle Ordner, in denen sich LESS-Dateien befinden, müssen zunächst dem Compiler hinzugefügt werden.

  2. Immer, wenn Sie eine Less-Datei im Editor bearbeiten, muss der Compiler im Hintergrund geöffnet sein.

  3. In Ihren HTML-Dateien binden Sie weiterhin wie gewohnt die automatisch erzeugten CSS-Dateien ein und nicht die Less-Dateien

  4. Die CSS-Dateien können auf wunsch auch komprimiert werden, da sie ja niemals direkt bearbeitet werden müssen

  5. Die LESS-Syntax ist eine Erweiterung der CSS-Syntax und voll abwärtskompatiel. Das bedeutet, jede gültige CSS-Datei ist auch gleichzeitig eine gültige LESS-Datei - nur eben ohne die Vorteile. Um also eine CSS-Datei nachträglich mit Less aufzupeppen, müssen Sie nur die Dateiendung von .css in .less ändern.

Kommentare in LESS

In Less sind - wie in Javascript - auch einzeilige Kommentare mit zwei Schrägstrichen // am Anfang möglich.

LESS-Kommentare mit // werden nicht in die kompilierte CSS-Datei übernommen. Kommentare mit /**/ bleiben hingegen erhalten. Letzteres ist z.B. nützlich, Letzters ist z.B. nützlich, wenn Sie in Ihre CSS-Dateien Copyright-Informationen integrieren möchten.
