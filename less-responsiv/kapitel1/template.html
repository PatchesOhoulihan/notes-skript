Die Limitationen von CSS

Durch die immer umfangreicheren Funktionen von CSS 3 werden auch einige Limitationen immer deutlicher, die die Arbeit mit CSS erschweren und unnötig umständlich machen. Solche Limitationen sind:

  1. In CSS sind keine Variablen möglich

    Stellen Sie sich vor, Sie haben für Ihre Website eine bestimmte Akzentfarbe, z.B. #f60. Wenn Sie diese Farbe verchiedenen Elementen auf Ihrer Seite - etwa Textbereichen oder Konturen - zuweisen möchten, müssen Sie sie jedesmal erneut notieren. Sie müssen bei einer Änderung diese Farbe an allen Stellen in Ihrem CSS-dokument einzeln ändern! Wäre es nicht schön, wenn Sie die Farbe nur einmal an einer zentralen Stelle in Ihrem Dokument als Variable definieren könnten? Bei einer Änderung der Variablen würde die Farbe in Ihrem gesamten CSS-Dokument automatisch angepasst werden

  2. CSS-Selektoren lassen sich nicht verschachteln

    Bei dem Descendant Selector müssen Sie jedesmal den kompletten Pfad notieren, z.B

    #header{irgendwas}

    #header nav{irgendwas}

    #header nav ul{irgendwas}

    #header nav ul li{irgendwas}

    #header nav ul li a{irgendwas}

    #header nav ul li a:hover{irgendwas}

    Hier würde es enorm viel Tipparbeit sparen, wenn Sie die Selektoren - analog zu HTML - verschachteln im Grunde also das DOM direkt abbilden könnten

  3. Klassen lassen sich nicht andere Selektoren als Eigenschaft zuweisen

    Es kommt häufig vor, dass Sie verschiedene Elementen auf Ihrer Seite immer wieder dieselben Eigenschaften zuweisen möchten. Wäre es nicht enorm praktisch, wenn Sie diese Eigenschaften zentral in einer Klasse speichern und diese Klasse dann ihrerseits den verschiedenen Elementen zuweisen zu könnten

    Dies alles ist möglich. Es gibt dafür schon seit geraumer Zeit gleich mehrere konkurrierende Technologien, sogenannte CSS Preprocessoren. Die beiden bekanntesten sind Less und SCSS. Die beiden unterscheiden sich mittlerweile fast nur noch in Syntaktischen Details.

Was ist LESS

LESS-Dateien können genauso wie CSS-Dateien in jedem beliebigen Editor erzeugt werden. Die Syntax von LESS erweitert die Syntax von CSS um die oben genannten Funktionen. So können Sie in LESS folgendes schreiben, um wie in Beispiel 1 eine Variable für eine globale Farbe zu definieren.

@meine_farbe: #f60

h1{color: @meine_farbe}

header{border: 1px solid @meine_farbe}

In Zukunft müssen Sie nur noch den Wert der Variablen @meine_farbe ändern, um diese Farbe in Ihrem gesamten Dokument anzupassen.

#header{
  nav{
    ul{
    }
  }
}

Die Verschachtelung der Selektoren entspricht exakt dem HTML-DOM und ist dadurch verständlicher, vor allem aber auch kürzer!

.box{
  border: 1px solid #666
  margin-bottom: 1rem;
  background-color:silver;}

  }
