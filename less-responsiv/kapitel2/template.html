Kapitel 2 Variablen.

Variablen definieren

In CSS müssen Sie den Hex-Wert für die Farbe jedesmal, wenn Sie sie verwenden, von neuem notieren. Vor allem aber müssen Sie sie bei einer etwailigen Änderung an jeder Stelle in Ihrem CSS-Dokument ändern. Dies ist aufwendig, mühsam und vorallem fehleranfällig.

Mit LESS können Sie diese Farbe an zentraler Stelle in Ihrem Dokument einmalig als Variable anlegen und dieser dann beliebig vielen Eigenschaften als Wert zuweisen Wenn Sie eine Farbe ändern möchten, ändern Sie nur den Wert der Variablen. Der LESS-Compiler erzetzt dann überall im resultierenden CSS-Dokument dynamisch die Varibale durch den entsprechenden Farbwert.

Da sich LESS so eng wie möglich an die "echte" CSS-Syntax hält, werden Variablen gebildet @-Regeln.

@fontColor: #333

Mehrere Werte für eine Variable

Genauso wie CSS-Eigenschaften können LESS-Variablen auch mehrere Werte enthalten

BSP: @basefront: "Lucida Grande", "Lucida Sans Unicode", "Lucida Sans", Verdana, Helvetica, Arial, sans-serif

Auch folgender Code wäre möglich.

BSP: @myShadow: 2px 2px 4px rgba(0,0,0,0.4)

LESS-Variablen im Überblick:

  • Die Verwendung einer Variablen besteht immer aus zwei Schritten

    1. Deklaration: Das einmalige Anlegen der Variablen, i.d.R am Anfang des LESS-Dokuments

      @meineVariable: wert;

    2. Aufruf: Das beliebige häufige Zuweisen der Variablen als Wert für Eigenschaften.

      selektor{
      eigenschaft: @meineVariable;
      }

  • Varibalen sind frei wählbar, dürfen aber -wie immer- keine Leer oder Sonderzeichen enthalten. Sie können zur besseren Lesbarkeit halber in camelCase notieren: @diesIstEinLangerVariablenName

  • Variablen können theoretisch überall im Dokument stehen. Meist ist es aber sinnvoll, sie am Beginn des Dokuments zu notieren oder sogar in eine eigene Datei auszulagern

  • Beachten Sie jedoch: Wenn Sie eine Variable außerhalb von Selektoren anlegen, können Sie sie überall in Ihrem Projekt verwenden. Man spricht in diesem Fall von Globalen Variablen. Wenn Sie aber eine Variable innerhalb eines Selektors anlegen, gilt diese nur dort Außerhalb des Selektors ist diese Variable unbekannt. Solche Variablen werden lokale Variablen genannt

    @globaleVariable: #fff
    h1{
    @lokaleVariable #000
    }
    h2{
    @globalevariable//kann verwendet werden
    @lokaleVariable // ist unbekannt
    }

Mit Variablen rechnen

Der Clou: LESS kann auch rechnen! Sie können Variablen miteinander oder mit Zahlen addieren, subtrahieren, multiplizieren oder dividieren Dabei werden die jeweils verwendeten Maßeinheiten(px em % etc.) beibehalten.

Variationen

Zudem bietet LESS eine menge nützlicher Funktionen, Werte zu variieren

In unserer Übung haben wir eine Variable @linkColor mit dem Farbwert #f60 definiert. Diese Akzentfarbe soll nun auch als Hintergrundfarbe für den #content verwendet werden, ist dafür aber viel zu dunekl und zu stark gestättigt. LESS bietet die Möglichkeit, die Farbe aufzuhellen:

#content{
background-color: lighten(@linkColor,45%)
}

Der Ausdruck bedeutet "Helle die Farbe @linkColor um 45% auf". In der kompilierten CSS-Datei erscheint die Farbe als background-color: #fff0e5

Anstelle der Variablen @linkColor können Sie natürlich auch direkt einen Farbwert in die runden Klammern eingeben

LESS bietet eine Vielzahl solcher Funktionen

Farboperationen:

saturate(farbe, prozentwert) Sättigung erhöhen.
desaturate(farbe, prozentwert) sättigung vermindern
lighten(farbe, prozentwert) aufhellen
darken(farbe, prozentwert) abdunklen fade(farbe, prozentwert) Deckkraft verringern(regibt RGBA-Werte) grayscale(farbe) in Graustufen konvertiern.

Andere Funktionen

convert(wert, maßeinheit) rechnet einen Wert von einer Maßeinheit in eine andere um z.b convert(9s, ms)

unit(wert) oder Unit(wert, maßeinheit) ändert bzw. entfernt die Maßeinheit hinter einem Wert(ohne Umrechnung)
bsp. unit(5em) = 5
unit(5,px) = 5px

Maskieren unbekannter Syntax

Es kann im Einzelfall vorkommen, dass Sie Syntay verwenden müssen, die der LESS-Compiler nicht versteht, weil sie noch nicht dem CSS Standard entspricht. Dann schlägt die kompilierung fehl und Sie erhalten eine Fehlermeldung.

In diesem Fall haben Sie die Möglichkeit, den betreffenden Codebereich mit einer tilde ~ am Anfang sowie Anführungszeichen zu maskieren

LESS:

filter: ~"ms:hasalwaysItsownstyle"

Der so gekennzeichnete Codebereich wird unbesehen in die CSS-Datei übernommen

CSS:

filter:mshasalwaysItsownstlye

Verwende von Variablen in maskierten Strings

Trotzdem können Sie in solchen Strings Variablen verwenden. Diese werden dort mit geschweiften Klammern gekennzeichnet: @{variable}. Sie werden trotz der Maskierung ausgelesen:

@meinText: "Stuff"
filter: ~"ms:irgendwasblabla{meinText}"
