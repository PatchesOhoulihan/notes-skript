Kapitel 10 Media Queries.

Wann werden Media Queries benötigt

Eingangs wurde bereits erwähnt, das Weblayouts, die auf Desktop-Monitoren gut aussehen, für Smartphones häufig nicht geeignet sind. Dies kann unterschiedliche Gründe haben, die "üblichen Verdächtigen" hierfür sind jedoch Elemente, die nebeneinander angeordnet sind, wie horizontale Menüleisten oder mehrspaltige Layouts. Auf Smartphones - vor allem im Hochformat - werden Elemente sinnvollerweise eher untereinander angeordnet.

In solchen Fällen kommen wir mit Breitenangaben in Prozent nicht mehr weiter - hierfür benötigen wir umfassendere Layoutanpassungen. Dafür stehen in CSS ide Media Queries(Medienabfragen) zur Verfügung. Damit können Sie für verschiedene Bildschirm bzw. Viewport Größen unterschiedliche Layouts definieren.

Medientypen und Media Queries

Sie kennen bereits die Medientypen, die Sie bei beim Einbinden von CSS im head Ihrer HTML-Dateien angeben können.

<link href="styles.css" rel="stylesheet" type="text/css" media="screen"/>

Damit können Sie angeben, dass die eingebundene CSS-Datei nur für die Bildschirmdarstellung gilt(im Gegensatz etwa zum Druck, media="print")

Dieselbe Angabe ist alternative dazu auch mit CSS möglich:

@media screen{
  //Eigenschaften;
}

Innerhalb der geschweiften Klammern stehen dann die Selektoren un Eigenschaften, die nur für die Bildschirmdarstellung gelten, z.B

@media screen{
  body{
    background-color: black;
  }}

Dabei bezieht sich jedoch die Angabe screen - egal ob in HTML oder CSS notiert - auf alle Arten von Bildschrimen, sowohl auf Desktop-Monitore als auch auf Smartphone-Displays(Die Angabe media="handheld" bezog sich auf klassische Mobiletelefone u dgl. - meist noch mit monochromen Bildschirmen - und ist heute veraltet. Smartphones und Tablets hingegen gelten ebenfalls als Computerbildschirme screen). Deshalb wurden in CSS3 zusätzliche Unterscheidungsmöglichkeiten u.a anhand der Bildschirmgröße, eingeführt, die Media Queries. Auch diese können Sie entweder in HTML oder CSS notieren

HTML:

< link href="styles.css" rel="stylessheet" type="text/css" media="screen and (max-device-width: 1034px)" />

CSS:

@media screen and (max-device-width: 1034px){
  //Eigenschaften;
}

Hier wird der medientyp screen mit bestinnten Merkmalen(sogennannten Media Features hier die Maximalbreite) verknüpft. die im Beispiel angegenben CSS-Vorgaben gelten also nur für Bildschirme mit einer Maximalbreite von 1034px, nicht aber für größere Monitore

NOTE: Bisweilen sieht man hier noch den Zusatz only

@media only screen and(max-device-width: 1034)

Das Schlüsselwort only dient lediglich dazu, die Angaben vor älteren Browsern zu verbergen. Es ist heute kaum mehr notwendig.

HTML oder CSS

Der Unterschied, ob Sie die media Queries in die Verknüpfung im head Ihrer HTML-Datei oder in die CSS-Datei notieren, liegt auf der Hand: Im ersten Fall gelten die Vorgabn der gesamten verknüpften CSS-Datei nur für eine Bestimmte Bildschirmgröße. Sie bräuchten dann für jede Bildschirmgröße eine separate CSS-Datei.

Im zweiten Fall können Sie in einer einzigen CSS-Datei in beliebig vielen @media-Blöcken unterschiedliche Vorgaben für verschiedene Bildschirmgrößen machen. da in diesem Fall nur ein CSS-Dokument geladen werden muss, werden auch die HTTP-Requests verringert und dadurch die Ladezeiten beschleunigt

Ich empfehle Ihnen deshalb folgende Vorgehensweise: Notieren Sie die einfachen Medientypen weiterhin in die HTML-Verknüpfung, z.B

<link href="screen.css" rel="stylesheet" type="text/css" media="screen" >

<link href="print.css" rel="stylesheet" type="text/css" media="print" >

In die screen.css kommen dann die alternativen Vorgaben für verschiedene Bildschirmgrößen. Dort ist die Angabe screen nun nicht mehr notwendig und kann weggelassen werden. Bleibt noch folgende Syntax:

@media (max-device-width: 1024px){
  //Vorgaben}

@media (max-device-width: 800px){
  //Vorgaben}

Bildschirmgröße und Viewportgröße

Mit dieser Syntax haben Sie nun die Möglichkeit, in einer einzigen CSS-Datei für jedes beliebige Element Ihrer Website beliebig viele Layout- und Designvarianten zu definieren. Dabei haben Sie die Wahl, ob Sie dafür die Bildschirmgröße oder die Viewport-größe zugrundelegen.

Folgende Größenangaben stehen u.a zur Verfügung(die Pixelwerte sind natürlich nur Beispielwerte):

Größenangaben für den Viewport Größenangaben für den Bildschirm

@media(width: 1024px)
@media(min-width: 1024px)
@media(max-width: 1024px)

@media(device-width:: 1024px)
@media(min-device-width: 1024px)
@media(max-device-width: 1024px)

@media(height: 768px)
@media(min-height: 768px)
@media(max-height)

@media(device-height:768px)
@media(min-device-height: 768px)
@media(max-device-height: 768px)

Bildschirmgröße device

Die device-Varianten beziehen sich auf die Bildschirmgröße des jeweiligen Geräts(Computermonitor, Smartphone), unabhängig von der tatsächlichen Fenstergröße des Browsers. Dadurch haben diese Varianten jedoch zwei gravierende Nachteile:

  1. Sie können nicht direkt am Arbeitsrechner getestet werden.

  2. Sie unterscheiden nicht zwischen Hoch- und Querformat eines Mobilgerätes, d.h wenn Sie ein Mobiltelefon hochkant halten, ist die max-device-width trotzdem die Breite des Querformats.

    Für dieses Problem gäbe es die folgende Lösung:

    @media(max-device-width: 1024px) and (orientation: portrait){
      /*Vorgaben*/}

    @media(max-device-width: 1024px) and (orientation: landscape){
      /*Vorgaben*/}

    Damit könnten Sie abfragen, ob der Benutzer das Gerät gerade im Hoch- oder im Querformat hält, und auch dafür unterschiedliche Layouts definieren. Dies bedeutet jedoch zusätzlichen Aufwand in unseren ohnehin schon recht komplexen CSS-Vorgaben.

Viewport-Größe

Ich empfehle Ihnen deshalb die Verwendung der Viewport-Varianten(ohne Device). Diese haben ensprechende zwei Vorteile:

  1. Sie können direkt am Arbeitsrechner durch einfaches Ändern der Breite des Browserfensters getestet werden.

  2. Sie reagieren somit auch automatisch auf einen Wechsel zwischen Hoch- und Querformat, da sie sich auf die tatsächliche, aktuelle Größe des Viewport - auch bei Mobilgeräten - beziehen, unabhängig von den Bildschirmabmessungen.

Verwendung von Media Queries: Breakpoints

Bei den Größenangaben orientieren Sie sich in der Regel natürlich primär an der Breite des Viewports und nicht an der Höhe. Dabei sind Sie auch in keiner Weise beschränkt auf bestimmte Standardauflösungen oder die Abmessungen bestimmter Geräte, sondern Sie definieren für jede Webseite individuell die verschiedenen Breiten, an denen sich das Layout ändern der Seite ändern soll. Diese sebstdefinierten Breitenangeaben nennt man acu h Breakpoints

In der Praxis hat sich folgendes Vorgehen bewährt:

  1. Sie definieren zunächst wie gewohnt(ohne Media Queries) in CSS/LESS sämtliche Layout und Designvorgaben Ihrer

  2. Dann ziehen Sie ganz einfach das Browserfenster keiner bis zu einer Breite, an der das Layout nicht mehr gut aussieht(wenn z.B textspalten zu schmal werden oder horizontalle Menüleisten umbrechen)

    NOTE: Zu diesem Zweck bieten die meisten modernen Browser nützliche Hilfsmittel an. So gibt es z.B in Firefox im Menü Extras ⇒ Web-Entwickler die Option Bildschirmgrößen testen. Dort können Sie komfortabel verschiedene Viewport-Größen ausprobieren und ablesen.

  3. Für diese Breite notieren Sie eine neue @media-Regel(Breakpoint) für die max-width und notieren dort sämtliche notwendigen Änderungen für sämtliche betroffenenen Selektoren(Eigenschaften, die sich nicht ändern sollen, müssen Sie nicht nochmals notieren). Verwendung der max-width gelten die Änderungen für alle Viewports, deren Breite kleiner oder gleich der angebenen ist.

  4. Dann ziehen Sie das Browserfenster noch kleiner bis zum nächsten Layout-Bruh und notieren für diese Breite den nächsten Breakpoint. Sie sollten dabei natürlich darauf achten, so viel wie nötige, aber so wenige wie möglich zu setzen.

NOTE: Viele Designer bevorzugen heutzutage den umgekehrten Weg nach dem Prinzip Mobile First. Dabei wird zunächst eine minimaltistische Version des Layouts für Mobilegeräte entwickelt und erst danach für größere Bildschirme erweitert. Dieses Vorgehen hat jedoch den Nachteil, dass die Desktop-Varianten oftmals wie enorm vergrößerte SmartphoneWebsites mit riesigen Schriften und spärlichen Inhalten aussehen. Auch ist diese Variante für Anfänger weniger gut geeignet.

Ein einfaches Beispiel: Ihr HTML-Dokument enthält drei <article>-Elemente, die Sie in CSS mit float:left nebeneinander gesetzt haben

Wenn Sie das Browserfenster zusammenschieben, stellen Sie fest, dass die Boxen bei einer Breite von 800px zu schmal werden. Sie beschließen also, dass die Boxen ab dieser Breite nicht mehr nebeneinander, sondern untereinader stehen sollen:

Das CSS dafür könnte so aussehen:

article{
  background-color: silver;
  width:30%;
  margin-right: 3%;
  float: left;}

@media(max-width: 800px){
  article{
    width: auto;
    margin 00 1rem 0;
     float: none;
  }
}

Einbinden von Media Queries

Für die Einbindunug von Media Queries gibt es mehrere Möglichkeiten. Dies ist zum einen abhängig vom Umfang der CSS Vorgaben und zum anderen davon, ob Sie Less verwenden.

  1. Für kleinere Projekte mit nur einer CSS-Datei können Sie die Media Queries zusammen ans Ende der CSS-Datei notieren.

  2. Für umfangreichere Projekte lohnt sich die Auslagerung der Media Queries zusammen in eine separate Datei. Diese wird dann im head der HTML-Datei oder in CSS/LESS mit @import eingebunden. Im Falle von Less werden die Media Queries dann zusammen mit den anderen Vorgaben in eine einzige CSS-Datei kompiliert.

  3. LESS erlaubt noch eine dritte, sehr elegante Variante: Damit können Sie die Media Queries in den jeweiligen Selektor verschachteln. Dies ist mit normalem CSS nicht möglich und hat den Vorteil, dass Sie alle Vorgaben für einen Selektor beieinander stehen haben. Unser obriges Beispiel mit den drei Boxen könnte dann so aussehen:

LESS:

aritcle{
  background-color: silver;
  width:30%;
  margin-right: 3%;   float: left;   @media(max-width: 800px){
    width: auto;
    margin: 0 0 1rem 0;     float: none;  }
}

Da Sie bei dieser Variante die Breitenangabe in jedem Selekor immer wieder aufs Neue verwenden, können Sie sie sogar in eine Variable auslagern. Diese muss allerdings mit einer Tilde am Anfang und Anführungszeichen maskiert werden.

@mobile: ~"(max-width: 800px)"

aritcle{
  background-color: silver;
  width:30%;
  margin-right: 3%;   float: left;   @media(max-width: 800px){
    width: auto;
    margin: 0 0 1rem 0;     float: none;  }
}

Allerdings kann diese Verschachtelung bei umfangreichen Vorgaben auch unübersichtlich werden. Wir werden deshalb in der folgenden Übung die zweite Variante nutzen und unsere Media Queries in eine separate Datei auslagern.
