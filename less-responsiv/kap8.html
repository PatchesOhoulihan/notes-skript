Kapitel 8 Keyframe-Animationen.

Das Grundprinzip von CSS-Animationen

Neben den bereits bekannten transitions können Sie in CSS 3 auch selbstlaufende Animationen erstellen, wie es bisher nur mit animierten GIFs, Flash oder Javascript möglich war. Dabei sind Sie nicht auf Bilder beschränkt, sondern Sie können jedes beliebige HTML-Element animieren.

Im Unterschied zu den transitions benötigen CSS-Animationen nicht zweinded ein Ereignis(z.B hover) als Auslöser, sondern sie können von alleine starten und endlos abgespielt werden.

Das Prinzip ist einfach: Bei einer CSS-Animation werden Eigenschaften über die Zeit hinweg verändert. So können Sie z.B die Eigenschaft opacity animieren, um ein Element ein- oder auszublenden, oder Sie ändern die Eigenschaften left oder right eines absolut positionierten Elements, um es horizontal üver die Webseite zu bewegen.

Ähnlich wie in anderen Animationsprogrammen müssen sie dabei nicht - wie etwa im Daumenkino oder einer klassischen Stop-motion-Animation(Trickfim)- jeden einzelen Bewegungsschritt explizit angeben, sondern Sie definieren nur die Momente im Verlauf der Animation, an denen sich etwas ändern soll. Im einfachsten Fall sind das lediglich der Anfangs- und Endpunkt der Animation.

Beispiel:

ein element soll von links nach rechts über die Seite wandern. In diesem Fall müssen Sie nur die Start- und die Endposition des Elements angeben, die Zwischenschritte werden vom Browser automatisch berechnet. Diese automatische Berechnung von Zwischenschritten nennt man auch Tweening.. Die manuell definierten Start- und Endpunkte der Animation nennt man Keyframes

Eine Animation wird in CSS also mit Hilfe von Keyframes erzeugt. Mit Keyframes definieren Sie, welche Eigenschaft zu welchem Zeitpunkt geändert werden soll.

Animation definieren

Die @keyframe-Regel

Zu diesem Zweck steht in CSS die @-Regel @keyframes zur Verfügung. Das bedeutet, eine Animation wird zunächst unabhängig von einer bestimmten Element oder Selektor definiert. Das hat den Vorteil, dass Sie die Animation danach beliebig vielen Elementen zuweisen können.

Ärgerlicherweise müssen Sie die keyframes zur Zeit wieder mit Herstellerpräfixen versehen, d.h Sie müssen jede Animation dreimal definieren.

Um eine auf diese Weise erstellte Animation eindeutig identifiezieren und anschließend einem Element zuweisen zu können, benötigen sie einen eindeutigen, selbstdefinierten Namen.

@-webkit-keyframes einblenden{
//Definitionen
}
@-moz-keyframes einblenden{
//Definitionen
}
@keyframes einblenden{
//Definitionen
}

Keyframes definieren

Danach folgt die Definition der einzelnen Keyframes(Animationsschritte). Dies geschieht entweder mit Prozentwerten(bezogen auf die verstriche Zeit) oder einfach mit den Schlüsselwörtern from und to Um z.B. ein Element langsam einzublenden, notieren Sie(jeweils in den drei herstellerspezifischen Varianten) entweder:

@keyframes einblenden{
  0%{
    opacity: 0;
  }
  100%{
    opacity: 100%;
  }
}

@keyframes einblenden{
  from{
    opacity: 0;
  }
  to{
    opacity: 100%;
  }
}

die Variante mit Prozentwerten hat den Vorteil, dass Sie nicht auf den Anfangs- und Endpunkt beschränkt sind, sondern beliebig viele Zwischenschritte einbauen können, an denen sich die Animation ändern kann.

Auch sind Sie natürlich nicht auf eine Eigenschaft pro Keyframe beschränkt, sondern Sie können in jedem Keyframe beliebig viele Eigenschaften auf einmal ändern. Um z.B. unser Elemente beim Einbinden auch gleichzeitig zu vergrößern

@keyframes einblenden{
  from{
    opacity: 0;
    transform:scale(0);
  }
  to{
    opacity: 100%;
    transform:scale(1);
  }
}

Animation zuweisen

Die Animation mit dem Namen huepfen kann nun beliebigen Selektoren zugewiesen werden. Dies geschieht mit der Eigenschaft animation, ebenfalls wieder mit Vendor Specific Prefixes. Die allgemeine Syntax lautet;

selektor{
  -webkit-animation: vorgaben
  -moz-animation: vorgaben
  animation: vorgaben
}

Neben dem Namen können Sie noch eine Menge weiterer Vorgaben für die Animation machen:

animation: name dauer beschleunigung verzögerung wiederholung richtung füllmodus play-pause

Tatsächlich ist die Eigenschaft animation eine Sammeleigenschaft, für folgende Einzel-Eigenschaften

Eigenschaft

Beschreibung

animation-name

Name, der @keyframes-Animation

animation-duration

Dauer, der Animation in Sekunden oder Millisekunden.

animation-timing-function

Beschleunigung oder Verlangsamung der Bewegung(Kurve)

animation-delay

Verzögerung, bevor die Animation beginnt.

Animation-iteration-count

Anzahl der Wiederholungen der Animation

animation-direction

Richtung, der Animation(vorwärts/rückwärts)

animation-fill-mode

Füllmodus Zustand des Elements, nachdem die Animation zu Ende ist.

animation-play-state

Play/Pause, Animation starten/anhalten.

Einzelbildanimation

Wie wir gesehen haben, können wir nahezu jedes beliebige CSS-Eigenschaft animieren(Ausnahme sind solche die per Definition keine Zwischenschritte besitzten(display:block), so auch die eigenschaft Background-position Damit ist es möglich, Einzelbildanimationen nach dem klassischen Trickfilm-Prinzip(und damit qualitativ höherwertige Alternativen zu GIF-Animationen zu erzeugen.) Der Schlüssel dazu sind - wie wir sehen werden - die timing-functions

Voraussetzung dafür ist ein Hintergrund-Sprite, das alle Einzelbilder der Animation(auch Frames genannt) enthält.

Der springenden Punkt ist die timing-function. Neben linear ease usw. gibt es auch noch die Anweisung, die Animation in Sprüngen oder Einzelschritten abzuspielen Die Funktion lautet steps(), wobei Sie in die runden Klammern die Anzahl der einzelschritte notieren müssen.

Beispiel

.animation(bewegen 1s steps(8,end) infinite)

Unsere Animation besteht aus 9 Einzelbilder, wobei das erste und das letzte identisch sind. Da in diesem Fall der Übergang vom letzten zurück zum ersten Bild eine sichbare Verzögerung in der Animation bewirken würde, muss eines der beiden ausgelassen werden. Deshalb notieren wir nur 8 steps und bestimmen mit dem zweiten Wert, welches der Beiden Bilder -das erste(start) oder das letzte(end)- enterst werden soll.
