Kapitel 7 Grafische Effekte.

Trivia

In CSS-3 wurden zahlreiche neue Funktionen eingeführt, mit denen Sie Elemente auf ähnliche Weise manipulieren können, wie Sie es bisher nur von Grafik- oder Bildbearbeitungsprogrammen kennen. Dazu gehören z.B Transformation(Formänderung), masken und Filter. Dabei sind die Manipulationen natürlich nicht auf Bilder beschränkt, sondern können auf nahezu beliebige HTML-Elemente angewendet werden.

Beispiele und Übungen dazu finden Sie gesammelt im Ordner css3_advanced im Begleitmaterial zu diesem Buch. Für eine einheitliche Gestaltung der Beispiele und Übungen wurden die Foramtierungen z.T in externe LESS-Dateien ausgelagert.(Ordner design im Ordner styles), die von mehreren Übungen gemeinsam benutzt werden. Diese Dateien sind für die Übungen nicht direkt relevant. Sie sollten jedoch in der Ordnerstruktur beibehalten.

Transformationen

Vielleicht kenn Sie bereits das Menü Transformatieren in Photoshop oder Illustrator. Die Formänderungen, die Sie damit auf Bilder und Grafiken anwenden können, stehen nun auch in CSS zur Manipulation von HTML-Elementen zur Verfügung: Rotieren(rotate),skalieren(scale),neigen(skew) und versetzen(translate). Die CSS-Eigenschaft dafür lautet sinnigerweise transform z.Z wieder mit Herstellerpräfix.

-webkit-transform:funktion(parameter)
-moz-transform:funktion(parameter)
transform:funktion(parameter)

Nach dem Doppelpunkt steht die gewünschte Art der Transformation, dahinter(ohne Leerzeichen) in runden Klammern die Parameter für die Transformation

Beispiele(Ohne Herstellerpräfix)

transform: rotate(45deg);, Rotation um 45 Grad
transform:scale(1.5);
transform:skewX(-30deg)Neigung um -30 Grad auf der x-Achse

Unterschiede zu anderen Eigenschaften

Hier stellt sich naturlich die Frage, was eine Transformation mit scale von einer Größenänderung mit width oder height unterscheidet. Die Unterschiede sind folgende:

  • Die Eigenschaft transform verändert ein Element nur virtuell und wirkt eher wie ein Effektfilter: Das Element wird zwar im Browser größer angezeigt, hat aber intern noch seine Originalgröße- etwa so, als würden Sie es durch eine Lupe betrachten. So wirkt sich scale auch nicht auf die umgebenden elemente aus.(Sie werden durch die Skalierung nicht verdrängt.)

  • Mit scale wird das Element mitsamt seinem Inhalt(einschließlich des Textes) skaliert. Sie können z.B. mit scaleX eine Element samt Text horizontal verzerren, was mit width nicht möglich ist.

Ähnliches gilt analog auch für translate im Vergleich zu einer Verschiebung z.B mit position:relative;

Den Transformationspunkt festlegen

Die Eigenschaft transform-origin in den drei browserspezifischen Varianten. Mit dieser Eigenschaft können Sie den Referenz bzw. Mittelpunkt für die Transformation festlegen.

Die allgemeine Syntax lautet:

transform-origin: x-Achse y-Achse;

Mögliche Werte sind:
x-Achse y-Achse

left
right
center
längenmaß(px em % usw)

top
bottom
center
längenmaß(px em % usw)

So würde etwa bei der Vorgabe

transform-origin: left top;
transform: scale(2);

ein Element ausgehend von der linken oberen Ecke um 200% nach rechts unten skaliert werden.

filter: filterfunktion(parameter);

Masken(Webkit)

Auch die masken funktionieren zur Zeit nur in Webkit-basierten Browsern Damit haben sie die Möglichkeit, ein HTML-Element mit einer beliebigen Form zu maskieren. d.h das Element ist nur noch innerhalb der Form zu sehen. Die Form fungiert gleichsam als Stanze, um das Element "auszuschneiden". Das Prinzip ist in etwa vergleichbar mit den Schnittmasken in Photoshop:

Bildmasken

Bei Bildmasken kann die Maskenform ein beliebiges externes Bild(Pixel- oder Vektorgrafik) oder ein CSS-Farbverlauf(gradient) sein. Im Falle einer Pixelgrafik oder eines Verlaufs muss das Bild allerdings transparente Bereiche enthalten, sodass für Bilder in der Praxis ausschließlich des PNG-Format in Frage kommt. Die Farbe spielt dabei keine Rolle.

Dabei ist der Vorteil von Pixelgrafiken oder Verläufen, dass sie auch weiche Übergänge von deckend nach transparent enthalten können. Der Vorteil von SVG-Grafiken ist ihre verlustfreie Skalierbarkeit

Bildmasken werden mit ganz ähnlichen eigenschaften eingebunden wie Hintergrundbilder:

-webkit-mask-image
-webkit-mask-position
-webkit-mark-repeat
-webkit-mask-attachment
-webkit-mask-size

Clipping

Ganz anders funktioniert das Clipping Damit können Sie lediglich Hintergrundbilder maskieren. Die Eigenschaft dafür lautet auch entsprechend background-clip

Eigentlich können Sie mit der Eigenschaft background-clip nur den Ausdehnungsbereich von Hintergrundbildern bestimmen. Sie erlaubt folgende Eigenschaften:

background-clip:

border-box, der Hintergrund erstreckt sich bis hinter die Kontur, standard einstellung
padding-box, der Hintergrund erstreckt sich zur Kontur, aber über das padding
content-box, der Hintergrund erstreckt sich nur bis zum Padding.

Dies funktioniert in allen modernen Browsern ohne Herstellerpräfix

Webkit-Browser erweitern diese Eigenschaft jedoch um folgenden Wert:

-webkit-background-clip:text
